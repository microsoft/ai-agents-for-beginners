# மாதிரி ரிசீட் கொண்டுதக்கங்கள்

நோட்புக் இயக்காமல் பரிசோதிக்கும் மூன்று முன்னறிமுக ரிசீட் கோப்புகள்.

| கோப்பு | அது என்ன |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` கருவி அழைப்பிற்கு செல்லுபடியாகும் கையொப்பமிடப்பட்ட ரிசீட். சரிபார்ப்பு True ஆகிறது. |
| `02_tampered_receipt.json` | கையொப்பமிடுகையின் பிறகு ஒரு புலம் மாற்றப்பட்ட அதே ரிசீட். சரிபார்ப்பு False ஆகிறது. |
| `03_chain_three_receipts.json` | மூன்று செல்லுபடியாகும் ரிசீட்டுகளின் சங்கிலி (search, hold, book) மற்றும் ஒவ்வொன்றையும் முன்னையதுடன் இணைக்கும் `previous_receipt_hash`. |

## மாதிரிகளை சரிபார்த்தல்

நோட்புக் சரிபார்ப்பை நான்கு பகுதிகளாக முன்னெடுக்கிறது. 
இந்த கொண்டுதக்கங்களை நோட்புக் கதையை இயக்காமே நேரடியாக சரிபார்க்க:

```python
import json
from pathlib import Path

# நீங்கள் இறக்குமதி மற்றும் உதவி செயல்பாடுகளை முடித்திருக்கிறீர்கள் என்று கருதப்படுகிறது
# 18-signed-receipts.ipynb இன் பிரிவுகள் 1 மற்றும் 2 இலிருந்து.

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # உண்மை

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # தவறு

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## இதைப் பெறுவது எப்படி

இந்த கொண்டுதக்கங்கள் நோட்புக்குக்கொடுக்கும் அதே குறியீடு பாதையைப் பயன்படுத்துகின்றன,
ஒரு நிலையான கையொப்ப சாவி மற்றும் பைட்-மீண்டும் உருவாக்கக்கூடிய கால அட்டவணைகளுடன். மறுவரைவு செய்ய:

```bash
python3 generate_fixtures.py
```

(இந்த ஸ்கிரிப்ட் இந்த அடைவின் `generate_fixtures.py` ஆகும்.)

## மாணவர்கள் மூல JSON ஐ ஆய்வு செய்வதிலிருந்து கற்றுக்கொள்ளும் போது

மூல ரிசீட் வடிவத்தை வாசிப்பது நோட்புக்கில் உள்ள செல்களைப்போல் அனைத்தும் சர்வகாலமாக வழங்காது என்ற உள்ளுறை வளர்க்கிறது. JSON ஐ விரைவாக பார்ப்ப மாணவர்கள் ஏதாவது கவனிக்கக்கூடும்:

1. கையொப்பம் ஒரு மறைந்த base64url வரி, ஆனால் பிற அனைத்து புலங்களும் தட்டச்சு வாசிக்கக்கூடிய JSON. கையொப்பம் உள்ளடக்கத்தை குறியாக்கம்செய்கின்றது அல்ல; அதனை நம்பிக்கை அளிக்கிறது.
2. `public_key` ரிசீட்டுக்குள் Embedded. ஒருவன் சரிபார்க்க வேறு தேவையில்லை (காவலர் அந்த சாவி உண்மையில் கோரியவர் சொந்தமானதாக இருக்கிறாரா என்பதையும் நம்புதல் உங்கள் பொறுப்பில் உள்ளது; இதற்கு README இல் அம்மாணிக்கை அடிப்படைகளைப் பார்க்கவும்).
3. எந்தவொரு புலத்தின் ஒரு எழுத்தையும் மாற்றியும் பின்னர் இந்தக் கோப்புடன் `02_tampered_receipt.json` ஐ ஒப்பிட்டால் பைட் நிலை செயல்முறை தெளிவாக விளங்கும்.
