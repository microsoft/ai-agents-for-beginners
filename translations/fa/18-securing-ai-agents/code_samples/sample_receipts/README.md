# نمونه رسیدها

سه فایل رسید از پیش تولید شده برای بررسی بدون اجرای دفترچه یادداشت.

| فایل | چیست |
|---|---|
| `01_valid_receipt.json` | یک رسید امضاشده معتبر برای فراخوانی ابزار `lookup_flights`. تأیید صحت بازگشت True دارد. |
| `02_tampered_receipt.json` | همان رسید با یک فیلد تغییر یافته پس از امضا. تأیید صحت بازگشت False دارد. |
| `03_chain_three_receipts.json` | زنجیره‌ای از سه رسید معتبر (جستجو، نگهداری، رزرو) با `previous_receipt_hash` که هر کدام را به قبلی متصل می‌کند. |

## تأیید نمونه‌ها

دفترچه یادداشت بررسی را در چهار بخش انجام می‌دهد. برای تأیید این نمونه‌ها
مستقیماً بدون اجرای متن دفترچه یادداشت:

```python
import json
from pathlib import Path

# فرض بر این است که واردات و توابع کمکی را تکمیل کرده‌اید
# از بخش‌های ۱ و ۲ از 18-signed-receipts.ipynb.

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # درست

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # نادرست

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## چگونه این‌ها تولید شده‌اند

نمونه‌ها از همان مسیر کد دفترچه یادداشت استفاده می‌کنند، با یک کلید امضای ثابت
و زمان‌های ثابت برای قابلیت بازتولید بایت به بایت. برای تولید مجدد:

```bash
python3 generate_fixtures.py
```

(اسکریپت در `generate_fixtures.py` در همین پوشه است.)

## دانش‌آموزان چه چیزی از بررسی فایل JSON خام می‌آموزند

خواندن قالب خام رسید شهودی را ایجاد می‌کند که سلول‌های دفترچه یادداشت
همیشه فراهم نمی‌کنند. دانش‌آموزانی که به فایل JSON نگاه می‌کنند اغلب متوجه می‌شوند:

1. امضا یک رشته مبهم base64url است، اما هر فیلد دیگر JSON ساده و قابل خواندن است. امضا محتوا را رمزنگاری نمی‌کند؛ بلکه به آن شهادت می‌دهد.
2. `public_key` در رسید تعبیه شده است. یک ممتحن چیزی جز این نیاز ندارد
   تا تأیید کند (مشروط به اینکه به این کلید اعتماد شود که واقعاً متعلق به صدورکننده ادعا شده باشد؛ به فایل README درس درباره زیرساخت هویت مراجعه کنید).
3. تغییر یک کاراکتر در هر فیلد، سپس مقایسه دوباره این فایل با
   `02_tampered_receipt.json`، مکانیزم سطح بایت را ملموس می‌سازد.
