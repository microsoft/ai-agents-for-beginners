# عوامل هوش مصنوعی برای مبتدیان - راهنمای مطالعه و خلاصه دوره

این راهنما خلاصه‌ای از دوره «عوامل هوش مصنوعی برای مبتدیان» را ارائه می‌دهد و مفاهیم کلیدی، چارچوب‌ها و الگوهای طراحی برای ساخت عوامل هوش مصنوعی را توضیح می‌دهد.

## ۱. مقدمه‌ای بر عوامل هوش مصنوعی

**عوامل هوش مصنوعی چیستند؟**  
عوامل هوش مصنوعی سیستم‌هایی هستند که قابلیت‌های مدل‌های زبان بزرگ (LLM) را با ارائه دسترسی به **ابزارها**، **دانش** و **حافظه** گسترش می‌دهند. بر خلاف یک ربات گفتگو مبتنی بر LLM که فقط متن تولید می‌کند، عامل هوش مصنوعی می‌تواند:  
- **محیط خود را درک کند** (از طریق حسگرها یا ورودی‌ها).  
- **استدلال کند** که چگونه یک مسئله را حل کند.  
- **عمل کند** تا محیط را تغییر دهد (از طریق اجرای ابزارها یا عملگرها).

**اجزای کلیدی یک عامل:**  
- **محیط**: فضای کاری که عامل در آن فعالیت می‌کند (مثلاً یک سیستم رزرو).  
- **حسگرها**: مکانیزم‌هایی برای جمع‌آوری اطلاعات (مثلاً خواندن یک API).  
- **عملگرها**: مکانیزم‌هایی برای انجام عملیات (مثلاً ارسال ایمیل).  
- **مغز (LLM)**: موتور استدلال که برنامه‌ریزی می‌کند و تصمیم می‌گیرد چه عملیاتی انجام شود.

## ۲. چارچوب‌های عامل‌محور

دوره از **Microsoft Agent Framework (MAF)** همراه با **Azure AI Foundry Agent Service V2** برای ساخت عوامل استفاده می‌کند:

| جزء | تمرکز | بهترین کاربرد |
|-----------|-------|----------|
| **Microsoft Agent Framework** | SDK یکپارچه پایتون/سی‌شارپ برای عوامل، ابزارها و جریان‌های کاری | ساخت عوامل با ابزار، جریان‌های کاری چندعاملی و الگوهای تولیدی. |
| **Azure AI Foundry Agent Service** | محیط اجرای ابری مدیریت‌شده | استقرار امن و مقیاس‌پذیر با مدیریت وضعیت، قابلیت رصد و اعتماد داخلی. |

## ۳. الگوهای طراحی عامل‌محور

الگوهای طراحی به ساختاردهی عملکرد عوامل کمک می‌کنند تا مسائل را به‌طور قابل اعتمادی حل کنند.

### **الگوی استفاده از ابزار** (درس ۴)  
این الگو به عوامل امکان تعامل با دنیای بیرون را می‌دهد.  
- **مفهوم**: به عامل یک «شِما» داده می‌شود (لیستی از توابع موجود و پارامترهایشان). LLM تصمیم می‌گیرد *کدام* ابزار را فراخوانی کند و با *چه* آرگومان‌هایی بر اساس درخواست کاربر.  
- **روند**: درخواست کاربر -> LLM -> **انتخاب ابزار** -> **اجرای ابزار** -> LLM (با خروجی ابزار) -> پاسخ نهایی.  
- **موارد استفاده**: بازیابی داده‌های زمان واقعی (آب و هوا، قیمت سهام)، انجام محاسبات، اجرای کد.

### **الگوی برنامه‌ریزی** (درس ۷)  
این الگو به عوامل توانایی حل وظایف پیچیده و چندمرحله‌ای می‌دهد.  
- **مفهوم**: عامل یک هدف کلی را به توالی‌ای از زیرکارهای کوچکتر تقسیم می‌کند.  
- **رویکردها**:  
  - **تقسیم‌بندی وظیفه**: تقسیم «برنامه‌ریزی سفر» به «رزرو پرواز»، «رزرو هتل»، «کرایه خودرو».  
  - **برنامه‌ریزی تکراری**: بازنگری برنامه براساس خروجی مراحل قبلی (مثلاً اگر پرواز پر باشد، انتخاب تاریخ دیگر).  
- **پیاده‌سازی**: اغلب شامل یک عامل «برنامه‌ریز» است که برنامه‌ای ساختاریافته (مثلاً JSON) تولید می‌کند که سپس توسط عوامل دیگر اجرا می‌شود.

## ۴. اصول طراحی

در طراحی عوامل، سه بُعد را در نظر بگیرید:  
- **فضا**: عوامل باید مردم و دانش را به هم متصل کنند، در دسترس اما کم‌جلب توجه باشند.  
- **زمان**: عوامل باید از *گذشته* بیاموزند، در *حال* نوآوری‌های مرتبط ارائه دهند و برای *آینده* سازگار شوند.  
- **هسته**: عدم قطعیت را بپذیرید اما از طریق شفافیت و کنترل کاربر اعتماد بسازید.

## ۵. خلاصه دروس کلیدی

- **درس ۱**: عوامل سیستم هستند، نه فقط مدل. آنها درک، استدلال و عمل می‌کنند.  
- **درس ۲**: Microsoft Agent Framework پیچیدگی فراخوانی ابزار و مدیریت وضعیت را انتزاع می‌کند.  
- **درس ۳**: طراحی با شفافیت و کنترل کاربر در ذهن انجام شود.  
- **درس ۴**: ابزارها «دست‌های» عامل هستند. تعریف شِما برای درک نحوه استفاده LLM حیاتی است.  
- **درس ۷**: برنامه‌ریزی «وظیفه اجرایی» عامل است که امکان کار روی جریان‌های کاری پیچیده را می‌دهد.
