# ਨਮੂਨਾ ਰਸੀਦ ਫਿਕਸਚਰਜ਼

ਨੋਟਬੁੱਕ ਨੂੰ ਚਲਾਏ ਬਿਨਾਂ ਨਿਰੀਖਣ ਲਈ ਤਿੰਨ ਪਹਿਲਾਂ-ਤਿਆਰ ਕੀਤੀਆਂ ਗਈਆਂ ਰਸੀਦ ਫਾਈਲਾਂ।

| ਫਾਈਲ | ਇਹ ਕੀ ਹੈ |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` ਟੂਲ ਕਾਲ ਲਈ ਇੱਕ ਵੈਧ ਸਾਈਨ ਕੀਤੀ ਰਸੀਦ। ਸੱਚਈ ਦੀ ਪੁਸ਼ਟੀ ਵਾਪਸ ਆਉਂਦੀ ਹੈ। |
| `02_tampered_receipt.json` | ਸਾਈਨ ਕਰਨ ਦੇ ਬਾਦ ਇੱਕ ਖੇਤਰ ਸੋਧਿਆ ਹੋਇਆ ਇੱਕੋ ਰਸੀਦ। ਸੱਚਈ ਦੀ ਪੁਸ਼ਟੀ ਗਲਤ ਵਾਪਸ ਆਉਂਦੀ ਹੈ। |
| `03_chain_three_receipts.json` | ਤਿੰਨ ਵੈਧ ਰਸੀਦਾਂ ਦੀ ਜੜੀ (search, hold, book) ਜਿੱਥੇ `previous_receipt_hash` ਹਰ ਇੱਕ ਨੂੰ ਪਿਛਲੀ ਨਾਲ ਜੋੜਦਾ ਹੈ। |

## ਨਮੂਨਿਆਂ ਦੀ ਸੱਚਾਈ ਦੀ ਜਾਂਚ

ਨੋਟਬੁੱਕ ਚਾਰ ਸੈਕਸ਼ਨਾਂ ਵਿੱਚ ਸੱਚਈ ਦੀ ਜਾਂਚ ਕਰਦਾ ਹੈ। ਇਹਨਾਂ ਫਿਕਸਚਰਾਂ ਨੂੰ ਸਿੱਧਾ ਨੋਟਬੁੱਕ ਕਹਾਣੀ ਸੋਝੀ ਬਿਨਾਂ ਜਾਂਚਣ ਲਈ:

```python
import json
from pathlib import Path

# ਇਹ ਮੰਨਦਾ ਹੈ ਕਿ ਤੁਸੀਂ ਇੰਪੋਰਟਾਂ ਅਤੇ ਸਹਾਇਕ ਫੰਕਸ਼ਨਾਂ ਨੂੰ ਪੂਰਾ ਕਰ ਲਿਆ ਹੈ
# 18-signed-receipts.ipynb ਦੇ ਭਾਗ 1 ਅਤੇ 2 ਤੋਂ।

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # ਸਹੀ

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # ਗਲਤ

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

ਅਤੇ ਬਾਈਟ-ਰੀਪ੍ਰੋਡਿਊਸਬਿਲਿਟੀ ਲਈ ਤਹਿ ਕੀਤੇ ਸਮਾਂ-ਚਿੰਨਾਂ ਨਾਲ। ਮੁੜਜਨਰੇਟ ਕਰਨ ਲਈ:

```bash
python3 generate_fixtures.py
```

(ਸਕ੍ਰਿਪਟ ਇਸ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ `generate_fixtures.py` ਤੇ ਹੈ।)

## ਬੱਚਿਆਂ ਨੂੰ ਕੱਚੇ JSON ਨਿਰੀਖਣ ਤੋਂ ਕੀ ਸਿੱਖਣ ਨੂੰ ਮਿਲਦਾ ਹੈ

ਕੱਚੇ ਰਸੀਦ ਫਾਰਮੈਟ ਨੂੰ ਪੜ੍ਹਨ ਨਾਲ ਉਹ ਸੋਝੀ ਬਣਦੀ ਹੈ ਜੋ ਨੋਟਬੁੱਕ ਦੀਆਂ ਸੈੱਲਾਂ ਹਮੇਸ਼ਾ ਨਹੀਂ ਦਿੰਦੀਆਂ। JSON ਨੂੰ ਝਲਕ ਨਾਲ ਦੇਖਣ ਵਾਲੇ ਬੱਚੇ ਅਕਸਰ ਨੋਟਿਸ ਕਰਦੇ ਹਨ:

1. ਦਸਤਖਤ ਇੱਕ opaque base64url ਸਟਰਿੰਗ ਹੁੰਦੀ ਹੈ, ਪਰ ਹਰ ਹੋਰ ਖੇਤਰ ਸਾਫ਼ ਪੜ੍ਹ੍ਹਨ ਯੋਗ JSON ਹੁੰਦਾ ਹੈ। ਦਸਤਖਤ ਸਮੱਗਰੀ ਨੂੰ ਗੁਪਤ ਨਹੀਂ ਕਰਦਾ; ਇਹ ਉਸਦੀ ਪੁਸ਼ਟੀ ਕਰਦਾ ਹੈ।
2. `public_key` ਰਸੀਦ ਵਿੱਚ ਜੁੜੀ ਹੁੰਦੀ ਹੈ। ਇੱਕ ਨਿਰੀਖਕ ਨੂੰ ਹੋਰ ਕਿਸੇ ਚੀਜ਼ ਦੀ ਲੋੜ ਨਹੀਂ ਹੁੰਦੀ ਜਾਂਚ ਲਈ (ਬਸ ਇਹ ਭਰੋਸਾ ਕਰਨ ਲਈ ਕਿ ਕੁੰਜੀ ਵਾਸਤਵ ਵਿੱਚ ਦਾਅਵਾ ਕੀਤੇ ਜਾਰੀ ਕਰਨ ਵਾਲੇ ਨਾਲ ਸੰਬੰਧਿਤ ਹੈ; ਵੇਖੋ ਪਾਠ README ਆਈਡੈਂਟੀਟੀ ਇੰਫ੍ਰਾਸਟਰੱਕਚਰ ਬਾਰੇ)।
3. ਕਿਸੇ ਵੀ ਖੇਤਰ ਦੇ ਇੱਕ ਅੱਖਰ ਨੂੰ ਸੋਧਣਾ, ਫਿਰ ਇਸ ਫਾਈਲ ਨੂੰ `02_tampered_receipt.json` ਨਾਲ ਮੁੜ-ਤੁਲਨਾ ਕਰਨਾ, ਬਾਈਟ-ਸਤਰ ਦੇ ਮਕੈਨੀਜ਼ਮ ਨੂੰ ਠੋਸ ਬਣਾਉਂਦਾ ਹੈ।
