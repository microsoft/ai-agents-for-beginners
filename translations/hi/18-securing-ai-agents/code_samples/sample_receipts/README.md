# नमूना रसीद फिक्स्चर

नोटबुक चलाए बिना निरीक्षण के लिए तीन पूर्व-जनरेटेड रसीद फ़ाइलें।

| फ़ाइल | क्या है यह |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` टूल कॉल के लिए एक मान्य साइन की गई रसीद। सत्यापन सच लौटाता है। |
| `02_tampered_receipt.json` | साइन करने के बाद एक फ़ील्ड संशोधित के साथ वही रसीद। सत्यापन असत्य लौटाता है। |
| `03_chain_three_receipts.json` | तीन मान्य रसीदों की एक श्रृंखला (search, hold, book) जिसमें `previous_receipt_hash` प्रत्येक को पूर्व की ओर लिंक करता है। |

## नमूनों का सत्यापन

नोटबुक चार अनुभागों में सत्यापन को समझाता है। सीधे इन फिक्स्चर का सत्यापन करने के लिए
नोटबुक कथा को चलाए बिना:

```python
import json
from pathlib import Path

# मानता है कि आपने आयात और सहायक फ़ंक्शन पूरे कर लिए हैं
# 18-signed-receipts.ipynb के अनुभाग 1 और 2 से।

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # सत्य

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # असत्य

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## कच्चे JSON का निरीक्षण करके छात्र क्या सीखते हैं

कच्चे रसीद प्रारूप को पढ़ना वह अंतर्दृष्टि बनाता है जो नोटबुक की कोशिकाएँ हमेशा प्रदान नहीं करतीं।
जो छात्र JSON को जल्दी देखते हैं, वे अक्सर ध्यान देते हैं:

1. हस्ताक्षर एक अस्पष्ट base64url स्ट्रिंग है, लेकिन हर अन्य फ़ील्ड सामान्य पठनीय JSON है।
   हस्ताक्षर सामग्री को एन्क्रिप्ट नहीं करता; यह उस पर प्रमाण देता है।
2. `public_key` रसीद में निहित है। एक ऑडिटर को सत्यापन के लिए कुछ और नहीं चाहिए
   (शर्त यह है कि कुंजी वास्तव में दावे किए गए जारीकर्ता की हो; पहचान इन्फ्रास्ट्रक्चर पर पाठ README देखें)।
3. किसी भी फ़ील्ड के एक अक्षर को संशोधित करना, फिर इस फ़ाइल की तुलना
   `02_tampered_receipt.json` के साथ करने पर, बाइट-स्तर की तंत्र को ठोस बनाता है।
