# AI एजंट्स नवशिक्यांसाठी - अभ्यास मार्गदर्शक आणि कोर्सचे सारांश

This guide provides a summary of the "AI Agents for Beginners" course and explains key concepts, frameworks, and design patterns for building AI Agents.

## 1. Introduction to AI Agents

**What are AI Agents?**
AI एजंट्स हे असे सिस्टम आहेत जे Large Language Models (LLMs) च्या क्षमतांना वाढवतात आणि त्यांना **साधने**, **ज्ञान**, आणि **स्मृती** यांचा प्रवेश देतात. पारंपारिक LLM चॅटबॉट जे फक्त प्रशिक्षण डेटावर आधारित मजकूर निर्माण करतो त्यापेक्षा वेगळे, एक AI एजंट हे करू शकते:
- **अनुभव घेणे** त्याचे पर्यावरण (सेन्सर्स किंवा इनपुटद्वारे).
- **तर्क** करून समस्या कशी सोडवावी हे ठरवणे.
- **क्रिया** करून पर्यावरण बदलणे (अॅक्ट्युएटर्स किंवा टूल अंमलबजावणीद्वारे).

**Key Components of an Agent:**
- **परिसर**: एजंट जेथे कार्य करतो ते ठिकाण (उदा., एक बुकिंग सिस्टम).
- **संवेदक**: माहिती गोळा करण्याची यंत्रणा (उदा., API वाचणे).
- **क्रियान्वयन यंत्रणा**: कृती पार पाडण्याची यंत्रणा (उदा., ईमेल पाठविणे).
- **Brain (LLM)**: योजना बनवणारा आणि कोणत्या क्रिया घ्यायच्या हे ठरवणारा तर्कशील इंजिन.

## 2. Agentic Frameworks

The course uses **Microsoft Agent Framework (MAF)** with **Azure AI Foundry Agent Service V2** for building agents:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | एजंट्स, टूल्स, आणि वर्कफ्लो साठी एकीकृत Python/C# SDK | टूल्ससह एजंट तयार करणे, मल्टी-एजंट वर्कफ्लो आणि उत्पादन पॅटर्न्स तयार करण्यासाठी. |
| **Azure AI Foundry Agent Service** | व्यवस्थापित क्लाउड रनटाइम | अंतर्निर्मित स्थिती व्यवस्थापन, निरीक्षणक्षमता आणि विश्वासासह सुरक्षित, स्केलेबल तैनाती. |

## 3. Agentic Design Patterns

Design patterns help structure how agents operate to solve problems reliably.

### **टूल वापरण्याचा पॅटर्न** (धडा 4)
This pattern enables agents to interact with the outside world.
- **संकल्पना**: एजंटला एक "schema" दिला जातो (उपलब्ध फंक्शन्स आणि त्यांच्या पॅरामीटर्सची यादी). LLM वापरकर्त्याच्या विनंतीनुसार ठरवतो *कोणता* टूल कॉल करायचे आणि *कोणत्या* अर्ग्युमेंट्ससह.
- **प्रवाह**: User Request -> LLM -> **Tool Selection** -> **Tool Execution** -> LLM (with tool output) -> Final Response.
- **उपयोग प्रकरणे**: रिअल-टाइम डेटा मिळवणे (हवामान, स्टॉक किंमती), गणितीय गणना करणे, कोड चालविणे.

### **योजना पॅटर्न** (धडा 7)
This pattern enables agents to solve complex, multi-step tasks.
- **संकल्पना**: एजंट एक उच्च-स्तरीय उद्दिष्ट लहान उप-कार्यांच्या अनुक्रमात विभागतो.
- **पद्धती**:
  - **Task Decomposition**: "प्रवासाची योजना" चे विभाजन "फ्लाइट बुक करणे", "हॉटेल बुक करणे", "कार भाड्याने घेणे" मध्ये करणे.
  - **Iterative Planning**: मागील चरणांच्या आउटपुटनुसार योजना पुन्हा तपासणे (उदा., जर फ्लाइट भरली असेल तर वेगळा दिनांक निवडणे).
- **अमलबजावणी**: हे सहसा एक "Planner" एजंट समाविष्ट करते जो JSON सारखी संरचित योजना तयार करतो आणि नंतर ती योजना इतर एजंटद्वारे अंमलात आणली जाते.

## 4. Design Principles

When designing agents, consider three dimensions:
- **Space**: एजंट्स लोकं आणि ज्ञान यांना जोडायला पाहिजेत, प्रवेशयोग्य पण असंख्यपणे विघ्न न आणणारे असायला पाहिजेत.
- **Time**: एजंट्सने *भूतकाळ* मधून शिका, *सध्याचा* काळात संबंधित सूचनादायक निर्देश द्या, आणि *भविष्य* साठी अनुकूल व्हा.
- **Core**: अनिश्चिततेला स्वीकारा पण पारदर्शकता आणि वापरकर्ता नियंत्रणाद्वारे विश्वास निर्माण करा.

## 5. Summary of Key Lessons

- **धडा 1**: एजंट्स हे फक्त मॉडेल्स नाहीत; ते सिस्टम आहेत. ते अनुभव घेतात, तर्क करतात, आणि क्रिया करतात.
- **धडा 2**: Microsoft Agent Framework हे टूल कॉलिंग आणि स्थिती व्यवस्थापनाच्या गुंतागुंतीला सारांशित करते.
- **धडा 3**: पारदर्शकता आणि वापरकर्ता नियंत्रण लक्षात ठेवून डिझाइन करा.
- **धडा 4**: टूल्स हे एजंटचे "हात" आहेत. Schema घोषणा LLM साठी ते कसे वापरायचे हे समजून घेण्यासाठी महत्त्वाची आहे.
- **धडा 7**: नियोजन हे एजंटचे "कार्यकारी कार्य" आहे, जे त्याला क्लिष्ट वर्कफ्लोज हाताळण्यास सक्षम करते.
