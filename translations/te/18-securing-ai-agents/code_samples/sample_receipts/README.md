# నమూనా రసీదు ఫిక్సర్లు

నోట్‌బుక్ ను నడపకుండానే తనిఖీ కోసం మూడు ముందుగానే రూపొందించిన రసీదు ఫైళ్లు.

| ఫైలు | ఇది ఏమిటి |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` టూల్ కాల్ కోసం సరైన సంతకం చేసిన రసీదు. ధృవీకరణ True ఇస్తుంది. |
| `02_tampered_receipt.json` | సంతకం చేసిన తర్వాత ఒక ఫీల్డ్ మార్చబడిన అదే రసీదు. ధృవీకరణ False ఇస్తుంది. |
| `03_chain_three_receipts.json` | మూడు సరైన రసీదుల గొలుసు (search, hold, book) `previous_receipt_hash` తో ప్రతి ఒకటి మునుపటి దానికి లింక్ చేస్తుంది. |

## నమూనాలను ధృవీకరించడం

నోట్‌బుక్ ధృవీకరణను నాలుగు భాగాలలో చూపిస్తుంది. నోట్‌బుక్ వివరణలోకి వెళ్లకుండా
ఈ ఫిక్సర్లు నేరుగా ధృవీకరించడానికి:

```python
import json
from pathlib import Path

# మీరు ఆమদానులు మరియు సహాయక ఫంక్షన్లు పూర్తి చేసారని అనుకుంటుంది
# 18-signed-receipts.ipynb యొక్క విభాగాలు 1 మరియు 2 నుండి.

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # నిజం

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # తప్పు

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## ఇవి ఎలా రూపొందించబడ్డాయి

ఫిక్సర్లు నోట్‌బుక్ వలే ఒకే కోడ్ మార్గాన్ని వాడతాయి, ఒక స్థిర సంతకం కీ
మరియు బైట్-పునరుత్పత్తికి నిర్ధారిత కాలముద్రలతో. తిరిగి సృష్టించడానికి:

```bash
python3 generate_fixtures.py
```

( స్క్రిప్ట్ ఈ డైరెక్టరీలో `generate_fixtures.py` లో ఉంది.)

## విద్యార్థులు రా JSON ను పరిశీలించి ఏమి నేర్చుకుంటారు

రావ రసీదు ఫార్మాట్ చదవడం నోట్‌బుక్ సెల్స్ ఎప్పుడూ అందించని అవగాహనను పెంచుతుంది.
JSON ను స్కిమ్ చేసే విద్యార్థులు సాధారణంగా గమనిస్తారు:

1. సంతకం ఒక అజ్ఞాత base64url స్ట్రింగ్ కానీ, మిగతా ప్రతీ ఫీల్డ్ సులభంగా
   చదవగల JSON. సంతకం కంటెంట్ ను గుప్తీకరించదు; దానిపై ప్రమాణం చెయ్యడం.
2. `public_key` రసీదులో నిక్షిప్తం. ధృవీకరించడానికి ఆడిటర్ కి మరో ఏమీ అవసరం లేదు
   (అవిన్ని వాడిన అందించిన ప్రామాణికతపై నమ్మకం వుండాలి; ఐడెంటిటీ ఇన్‌ఫ్రాస్ట్రక్చర్ గురించిన లెస్‌న README చూడండి).
3. ఏ ఫీల్డ్ లో ఒక అక్షరం కూడా మార్చడం, ఆ తర్వాత ఈ ఫైల్ ను `02_tampered_receipt.json` తో పునర్మిల్లించడం,
   బైట్-స్థాయి యంత్రాంగం స్పష్టంగా మరింత అర్థమవుతుంది.
