# Пример фајлова са рачунима

Три претходно генерисана фајла са рачунима за инспекцију без покретања бележнице.

| Фајл | Шта је то |
|---|---|
| `01_valid_receipt.json` | Важи потписани рачун за позив алата `lookup_flights`. Верификација враћа True. |
| `02_tampered_receipt.json` | Исти рачун са једним пољем измењеним након потписивања. Верификација враћа False. |
| `03_chain_three_receipts.json` | Ланац од три важећа рачуна (претрага, резервација, книга) са `previous_receipt_hash` који повезује сваки с претходним. |

## Провера примера

Бележница пролази кроз проверу у четири одељка. Да бисте директно проверили ове фајлове
без проласка кроз нарацију у бележници:

```python
import json
from pathlib import Path

# Претпоставља се да сте завршили увозе и помоћне функције
# из одељака 1 и 2 од 18-signed-receipts.ipynb.

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # Тачно

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # Нетачно

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## Како су ови фајлови генерисани

Фајлови користе исти код као и бележница, са фиксним кључем за потписивање
и фиксним временским ознакама ради репродукције бајт-бајт. За поновну генерацију:

```bash
python3 generate_fixtures.py
```

(Скрипта се налази у `generate_fixtures.py` у овом директоријуму.)

## Шта студенти уче из прегледа сировог JSON-а

Читање сировог формата рачуна гради интуицију коју ћелије у бележници
често не пружају. Студенти који прелистају JSON често примете:

1. Потпис је непрозирни base64url низ, али сва остала поља су обичан
   читљиви JSON. Потпис не шифрује садржај; он га потврђује.
2. `public_key` је уграђен у рачун. Ревизор не треба ништа друго
   за проверу (уз услов да верује да кључ заиста припада траженом
   издаваоцу; видети README лекције о инфраструктури идентитета).
3. Измена једног знака у било ком пољу, а затим упоређење овог фајла са
   `02_tampered_receipt.json`, чини механизам на нивоу бајтова јасним.
