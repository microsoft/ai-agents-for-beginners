# AI 代理入門 - 學習指南與課程總結

本指南提供「AI 代理入門」課程的總結，並解釋建立 AI 代理的關鍵概念、框架與設計模式。

## 1. AI 代理介紹

**什麼是 AI 代理？**  
AI 代理是擴展大型語言模型（LLM）能力的系統，賦予它們存取**工具**、**知識**和**記憶**的能力。與僅根據訓練資料生成文本的標準 LLM 聊天機器人不同，AI 代理可以：  
- **感知**其環境（透過感測器或輸入）。  
- **推理**如何解決問題。  
- **行動**以改變環境（透過執行器或工具執行）。

**代理的主要組件：**  
- **環境**：代理操作的空間（例如預訂系統）。  
- **感測器**：蒐集資訊的機制（例如讀取 API）。  
- **執行器**：執行行動的機制（例如發送電子郵件）。  
- **大腦（LLM）**：計劃並決定採取哪些行動的推理引擎。

## 2. 代理框架

本課程使用 **Microsoft Agent Framework (MAF)** 搭配 **Azure AI Foundry Agent Service V2** 來建構代理：

| 組件 | 焦點 | 適用範圍 |
|-----------|-------|----------|
| **Microsoft Agent Framework** | 統一的 Python/C# SDK，支持代理、工具與工作流程 | 建立具工具能力的代理、多代理工作流程與生產模式。 |
| **Azure AI Foundry Agent Service** | 受管雲端運行時 | 具備內建狀態管理、可觀察性與信任的安全可擴展部署。 |

## 3. 代理設計模式

設計模式有助於結構化代理的運作，確保持續可靠地解決問題。

### **工具使用模式**（第 4 課）  
此模式讓代理能與外界互動。  
- **概念**：代理會獲得一組「schema」（可用功能及其參數列表）。LLM 根據用戶請求決定*呼叫哪個工具*以及*使用何種引數*。  
- **流程**：用戶請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM（帶工具輸出）-> 最終回應。  
- **用途**：取得即時資料（天氣、股價）、執行計算、執行程式碼。

### **規劃模式**（第 7 課）  
此模式使代理能完成複雜多步驟任務。  
- **概念**：代理將高階目標拆解為一連串較小子任務。  
- **方法**：  
  - **任務分解**：將「安排行程」拆解成「訂機票」、「訂旅館」、「租車」。  
  - **反覆規劃**：根據前一步結果重新評估計劃（比如機票客滿就改訂其他日期）。  
- **實作**：通常涉及一個「規劃者」代理產生結構化計劃（如 JSON），再由其他代理執行。

## 4. 設計原則

設計代理時，考慮三個維度：  
- **空間**：代理應連結人與知識，易於存取且不會干擾。  
- **時間**：代理應從*過去*學習、在*當下*提供相關提醒，並為*未來*適應。  
- **核心**：接受不確定性，並透過透明度與用戶控制建立信任。

## 5. 主要課程總結

- **第 1 課**：代理是系統，不只是模型。它們會感知、推理且行動。  
- **第 2 課**：Microsoft Agent Framework 抽象化了工具呼叫和狀態管理的複雜性。  
- **第 3 課**：設計時須考慮透明度與用戶控制。  
- **第 4 課**：工具是代理的「手」，schema 定義對 LLM 理解如何使用工具至關重要。  
- **第 7 課**：規劃是代理的「執行功能」，使其能處理複雜工作流程。
