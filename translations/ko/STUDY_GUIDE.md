# 초보자를 위한 AI 에이전트 - 학습 가이드 및 강의 요약

이 가이드는 "AI Agents for Beginners" 코스의 요약을 제공하고 AI 에이전트를 구축하기 위한 핵심 개념, 프레임워크 및 디자인 패턴을 설명합니다.

## 1. AI 에이전트 소개

**AI 에이전트란 무엇인가?**
AI 에이전트는 대형 언어 모델(LLMs)의 기능을 확장하여 **도구**, **지식**, **메모리**에 접근할 수 있게 하는 시스템입니다. 학습 데이터에 기반해 텍스트만 생성하는 일반적인 LLM 챗봇과 달리, AI 에이전트는 다음을 할 수 있습니다:
- **환경을 인지**합니다 (센서 또는 입력을 통해).
- **문제를 해결하는 방법을 추론**합니다.
- **행동**하여 환경을 변화시킵니다 (액추에이터 또는 도구 실행을 통해).

**에이전트의 핵심 구성요소:**
- **환경**: 에이전트가 작동하는 공간 (예: 예약 시스템).
- **센서**: 정보를 수집하는 메커니즘 (예: API 읽기).
- **액추에이터**: 동작을 수행하는 메커니즘 (예: 이메일 전송).
- **두뇌 (LLM)**: 어떤 행동을 취할지 계획하고 결정하는 추론 엔진.

## 2. 에이전트 프레임워크

이 과정은 에이전트를 구축하기 위해 **Microsoft Agent Framework (MAF)**와 **Azure AI Foundry Agent Service V2**를 사용합니다:

| 구성요소 | 초점 | 적합한 용도 |
|-----------|-------|----------|
| **Microsoft Agent Framework** | 에이전트, 도구 및 워크플로우를 위한 통합 Python/C# SDK | 도구를 사용하는 에이전트, 다중 에이전트 워크플로우 및 프로덕션 패턴 구축에 적합. |
| **Azure AI Foundry Agent Service** | Managed cloud runtime | 내장된 상태 관리, 관측성 및 신뢰 기능을 갖춘 안전하고 확장 가능한 배포. |

## 3. 에이전트 디자인 패턴

디자인 패턴은 에이전트가 문제를 신뢰성 있게 해결하도록 작동 방식을 구조화하는 데 도움이 됩니다.

### **도구 사용 패턴** (Lesson 4)
이 패턴은 에이전트가 외부 세계와 상호작용할 수 있게 합니다.
- **개념**: 에이전트에는 "schema" (사용 가능한 함수와 그 매개변수 목록)가 제공됩니다. LLM은 사용자 요청에 따라 *어떤* 도구를 호출할지와 *무슨* 인수로 호출할지를 결정합니다.
- **흐름**: 사용자 요청 -> LLM -> **도구 선택** -> **도구 실행** -> LLM (도구 출력 포함) -> 최종 응답.
- **사용 사례**: 실시간 데이터 조회(날씨, 주가), 계산 수행, 코드 실행.

### **계획 패턴** (Lesson 7)
이 패턴은 에이전트가 복잡한 다단계 작업을 해결할 수 있게 합니다.
- **개념**: 에이전트는 상위 목표를 여러 개의 더 작은 하위 작업 시퀀스로 분해합니다.
- **접근 방법**:
  - **작업 분해**: "여행 계획"을 "항공편 예약", "호텔 예약", "자동차 렌트"로 분할합니다.
  - **반복적 계획**: 이전 단계의 출력에 따라 계획을 재평가합니다 (예: 항공편이 만석이면 다른 날짜를 선택).
- **구현**: 일반적으로 구조화된 계획(예: JSON)을 생성하는 "Planner" 에이전트를 포함하며, 이 계획은 다른 에이전트에 의해 실행됩니다.

## 4. 설계 원칙

에이전트를 설계할 때 세 가지 차원을 고려하세요:
- **공간(Space)**: 에이전트는 사람과 지식을 연결해야 하며, 접근 가능하지만 눈에 거슬리지 않아야 합니다.
- **시간(Time)**: 에이전트는 *과거*로부터 학습하고, *지금*에 관련된 적절한 알림을 제공하며, *미래*에 적응해야 합니다.
- **핵심(Core)**: 불확실성을 수용하되 투명성과 사용자 제어를 통해 신뢰를 구축하세요.

## 5. 주요 강의 요약

- **Lesson 1**: 에이전트는 단순한 모델이 아니라 시스템입니다. 이들은 인지하고, 추론하며, 행동합니다.
- **Lesson 2**: Microsoft Agent Framework는 도구 호출과 상태 관리의 복잡성을 추상화합니다.
- **Lesson 3**: 투명성과 사용자 제어를 염두에 두고 설계하세요.
- **Lesson 4**: 도구는 에이전트의 "손"입니다. 스키마 정의는 LLM이 도구를 사용하는 방법을 이해하는 데 중요합니다.
- **Lesson 7**: 계획은 에이전트의 "집행 기능"으로, 복잡한 워크플로를 처리할 수 있게 합니다.
