# AI ഏജന്റുകൾ തുടങ്ങുന്നവർക്ക് - പഠന ഗൈഡ് & കോഴ്സ് സംഗ്രഹം

ഈ ഗൈഡ് "AI Agents for Beginners" കോഴ്സിന്റെ സംഗ്രഹം നൽകുകയും ഏഐ ഏജന്റുകൾ നിർമ്മിക്കാനുള്ള പ്രധാന ആശയങ്ങൾ, ഫ്രെയിംവർക്കുകൾ, ഡിസൈൻ പാറ്റേണുകൾ വിശദീകരിക്കുകയും ചെയ്യുന്നു.

## 1. Introduction to AI Agents

**AI Agents എന്താണ്?**
AI ഏജന്റുകൾ വലിയ ഭാഷാ മോഡലുകളുടെ (LLMs) കഴിവുകൾ **തന്ത്രങ്ങൾ**, **ജ്ഞാനം**, மற்றும் **മെമ്മറി** എന്നിവയിലേക്ക് ആക്സസ് നൽകിക്കൊണ്ട് വിപുലമാക്കുന്ന സിസ്റ്റങ്ങളാണ്. പരിശീലന ഡാറ്റയുടെ അടിസ്ഥാനത്തിൽ മാത്രം ടെക്സ്റ്റ് സൃഷ്ടിക്കുന്ന ഒരു സാധാരണ LLM ചാറ്റ്ബോട്ടിനേക്കാൾ വ്യത്യസ്തമായി, ഒരു AI ഏജന്റ് കഴിവുള്ളത്:
- **ഗ്രഹിക്കുക** അതിന്റെ പരിസ്ഥിതി (സെൻസറുകൾ അല്ലെങ്കിൽ ഇൻപുട്ടുകൾ വഴി).
- **യുക്തിപരമായി ചിന്തിക്കുക** ഒരു പ്രശ്നം എങ്ങനെ പരിഹരിക്കാമെന്ന്.
- **പ്രവർത്തിക്കുക** പരിസ്ഥിതി മാറ്റാൻ (അക്ചുവേറ്ററുകൾ അല്ലെങ്കിൽ ടൂൾ നിർവഹണം വഴി).

**Key Components of an Agent:**
- **Environment**: ഏജന്റ് പ്രവർത്തിക്കുന്ന സ്ഥലം (e.g., a booking system).
- **Sensors**: വിവരങ്ങൾ ശേഖരിക്കാനുള്ള ക്രമീകരണങ്ങൾ (e.g., reading an API).
- **Actuators**: പ്രവർത്തനങ്ങൾ നിർവഹിക്കാൻ ഉള്ള ക്രമീകരണങ്ങൾ (e.g., sending an email).
- **Brain (LLM)**: ഏത് പ്രവർത്തനങ്ങൾ എടുക്കണമെന്ന് പദ്ധതി രൂപപ്പെടുത്തുകയും തീരുമാനിക്കുകയും ചെയ്യുന്ന ന്യായീകരണ എൻജിൻ.

## 2. Agentic Frameworks

കോഴ്സ് ഏജന്റുകൾ നിർമ്മിക്കാൻ **Microsoft Agent Framework (MAF)**-നും **Azure AI Foundry Agent Service V2**-നുമായി ഉപയോഗിക്കുന്നു:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | ഏജന്റുകൾ, ടൂളുകൾ, വർക്‌ഫ്ലോകൾക്കുള്ള ഏകീകൃത Python/C# SDK | ടൂളുകളോടുകൂടെ ഏജന്റുകൾ നിർമ്മിക്കൽ, മൾട്ടി-ഏജന്റ് വർക്‌ഫ്ലോകൾ, ഉത്പാദന പാറ്റേണുകൾ. |
| **Azure AI Foundry Agent Service** | മാനേജ് ചെയ്ത ക്ലൗഡ് റൺടൈം | ഇൻബിൽറ്റ് സ്റ്റേറ്റ് മാനേജ്‌മെന്റ്, നിരീക്ഷണക്ഷമത, വിശ്വാസം എന്നിവയോടെ സുരക്ഷിതവും സ്കെയ്ലബിൾ ആയ ഡിപ്ലോയ്മെന്റ്. |

## 3. Agentic Design Patterns

ഡിസൈൻ പാറ്റേണുകൾ ഏജന്റുകൾ പ്രശ്നങ്ങൾ വിശ്വസനീയമായി പരിഹരിക്കാൻ എങ്ങനെ പ്രവർത്തിക്കണം എന്നു ഘടന നൽകാൻ സഹായിക്കുന്നു.

### **ടൂൾ ഉപയോഗ പാറ്റേൺ** (Lesson 4)
ഈ പാറ്റേൺ ഏജന്റുകൾക്ക് ബഹിരാകാശവുമായി ഇടപഴകാൻ അനുവദിക്കുന്നു.
- **Concept**: ഏജന്റിന് ഒരു "schema" (ലഭ്യമായ ഫംഗ്ഷനുകളുടെ പട്ടികയും അവയുടെ പാരാമീറ്ററുകളും) നൽകുന്നു. LLM ഉപയോക്താവിന്റെ അഭ്യർത്ഥനയെ അടിസ്ഥാനമാക്കി *ഏത്* ടൂൾ വിളിക്കണമെന്ന്, അതിനുള്ള *എന്ത്* ആർഗ്യുമെന്റുകളാണെന്ന് തീരുമാനിക്കുന്നു.
- **Flow**: ഉപയോക്തൃ അഭ്യർത്ഥന -> LLM -> **ടൂൾ തിരഞ്ഞെടുപ്പ്** -> **ടൂൾ നിർവഹണം** -> LLM (ടൂൾ ഔട്ട്പുട്ടിനോട് കൂടെ) -> അന്തിമ മറുപടി.
- **Use Cases**: റിയൽ-ടൈം ഡാറ്റ വീക്ഷിക്കൽ (കാലാവസ്ഥ, സ്റ്റോക്ക് വിലകൾ), കണക്കുകൂട്ടലുകൾ നിർവഹിക്കൽ, കോഡ് എക്സിക്യൂട്ട് ചെയ്യൽ.

### **Planning Pattern** (Lesson 7)
ഈ പാറ്റേൺ ഏജന്റുകൾക്ക് സങ്കീർണ്ണവും ഒരുাধিক-ഘട്ടവുമായ ജോലികൾ പരിഹരിക്കാൻ സഹായിക്കുന്നു.
- **Concept**: ഏജന്റ് ഒരു ഉയർന്ന நிலையில் ഉള്ള ലക്ഷ്യം ചെറിയ ഉപകാര്യങ്ങളായി വിഭജിക്കുന്നു.
- **Approaches**:
  - **Task Decomposition**: "ഒരു യാത്രാ പദ്ധതി" എന്നത് "ഫ്ലൈറ്റ് ബുക്ക് ചെയ്യുക", "ഹോട്ടൽ ബുക്ക് ചെയ്യുക", "കാർ വാടകയ്ക്ക് എടുക്കുക" എന്നതിലായി വിഭജിക്കൽ.
  - **Iterative Planning**: മുൻപുള്ള ഘട്ടങ്ങളുടെ ഔട്ട്പുട്ട് അടിസ്ഥാനമാക്കി പദ്ധതിയെ പുനഃമൂല്യവിചാരം ചെയ്യൽ (ഉദാ., ഫ്ലൈറ്റ് പൂർത്തിയെങ്കിൽ വ്യത്യസ്ത തിയതി തിരഞ്ഞെടുക്കുക).
- **Implementation**: പലപ്പോഴും ഒരു "Planner" ഏജന്റ് ഘടനാപരമായ ഒരു പദ്ധതി (e.g., JSON) 생성ചെയ്യുകയും അത് മറ്റ് ഏജന്റുകൾ നിർവഹിക്കുകയും ചെയ്യുന്നു.

## 4. Design Principles

ഏജന്റുകൾ രൂപകൽപ്പന ചെയ്യുമ്പോൾ മൂന്ന് பரിമാണങ്ങൾ പരിഗണിക്കുക:
- **Space**: ഏജന്റുകൾ ആളുകളെയും അറിവിനെയും ബന്ധിപ്പിക്കണം, ലഭ്യമാകണം പക്ഷേ അഭ്യന്തരമായിരിക്കണം.
- **Time**: ഏജന്റുകൾ *ഭൂതകാലത്തിൽ* നിന്ന് പഠിക്കണം, *ഇപ്പോൾ* പ്രസക്തമായ നുദ്ധുകൾ നൽകണം,യും *ഭാവിക്കായി* ഉടനടി അനുസരിക്കാൻ കഴിവുള്ളവയാകണം.
- **Core**: അനിശ്ചിതത്വത്തെ സ്വീകരിക്കുക, പക്ഷേ पारദർശിത്വവും ഉപയോക്തൃ നിയന്ത്രണവും വഴി വിശ്വാസം സ്ഥാപിക്കുക.

## 5. Summary of Key Lessons

- **Lesson 1**: ഏജന്റുകൾ സിസ്റ്റങ്ങളാണ്, പകുതി മോഡലുകൾ മാത്രം അല്ല. അവ ഗ്രഹിക്കുകയും, യുക്തിപരമായി ചിന്തിക്കുകയും, പ്രവർത്തിക്കുകയും ചെയ്യുന്നു.
- **Lesson 2**: Microsoft Agent Framework ടൂൾ കോളിംഗ് এবং സ്റ്റേറ്റ് മാനേജ്മെന്റിന്റെ സങ്കീർണത കൈവിടുന്നു.
- **Lesson 3**: പാർദർശിത്വവും ഉപയോക്തൃ നിയന്ത്രണവും മനസ്സിലാക്കി ഡിസൈൻ ചെയ്യുക.
- **Lesson 4**: ടൂളുകൾ ഏജന്റിന്റെ "കൈകളാണ്". Schema നിർവചനമാണ് LLM-ന് അവ ഉപയോഗിക്കുന്നത് മനസ്സിലാക്കാൻ നിർണായകം.
- **Lesson 7**: പ്ലാനിംഗ് ഏജന്റിന്റെ "എക്‌സിക്യൂട്ടീവ് ഫംഗ്ഷൻ" ആണ്, ഇത് സങ്കീർണ്ണ വർക്‌ഫ്ലോകൾ കൈകാര്യം ചെയ്യാൻ അവനെ സാധ്യമാക്കുന്നു.
