# AI 代理人入門 - 學習指南與課程摘要

本指南提供「AI Agents for Beginners」課程的摘要，並說明用於構建 AI 代理人的關鍵概念、框架與設計模式。

## 1. AI 代理人簡介

**什麼是 AI 代理人？**
AI 代理人是透過賦予大型語言模型 (LLMs) 對 **工具**、**知識** 與 **記憶** 的存取權來擴展其能力的系統。與僅根據訓練數據產生文字的標準 LLM 聊天機器人不同，AI 代理人可以：
- **感知** 其環境（透過感測器或輸入）。
- **推理** 如何解決問題。
- **行動** 改變環境（透過執行工具或致動器）。

**代理人的關鍵組成：**
- **環境**：代理人所運作的空間（例如：訂票系統）。
- **感測器**：蒐集資訊的機制（例如：讀取 API）。
- **致動器**：執行動作的機制（例如：發送電子郵件）。
- **大腦 (LLM)**：規劃與決定要採取哪些行動的推理引擎。

## 2. 代理框架

課程使用 **Microsoft Agent Framework (MAF)** 與 **Azure AI Foundry Agent Service V2** 來構建代理人：

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | 用於代理人、工具與工作流程的統一 Python/C# SDK | 建構具有工具、多代理工作流程與生產模式的代理人。 |
| **Azure AI Foundry Agent Service** | 託管的雲端執行時 | 提供安全、可擴展的部署，並內建狀態管理、可觀測性與信任。 |

## 3. 代理設計模式

設計模式有助於結構化代理人的運作方式，以可靠地解決問題。

### **工具使用模式** (Lesson 4)
此模式讓代理人能與外部世界互動。
- **概念**：代理人會被提供一個 "schema"（列出可用函數及其參數的清單）。LLM 會根據使用者的請求決定 *呼叫哪個* 工具以及使用 *哪些* 引數。
- **流程**：使用者請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM (含工具輸出) -> 最終回應。
- **使用情境**：擷取即時資料（天氣、股票價格）、執行計算、執行程式碼。

### **規劃模式** (Lesson 7)
此模式讓代理人能解決複雜、多步驟的任務。
- **概念**：代理人將高階目標拆解為一連串較小的子任務。
- **方法**：
  - **任務分解**：將 "Plan a trip" 拆成 "Book flight"、"Book hotel"、"Rent car"。
  - **反覆規劃**：根據先前步驟的輸出重新評估計畫（例如：若班機客滿，則選擇不同日期）。
- **實作**：通常會包含一個「Planner」代理人，產生結構化計畫（例如 JSON），然後由其他代理人執行。

## 4. 設計原則

在設計代理人時，考慮三個面向：
- **空間**：代理人應連結人與知識，易於取得但不打擾使用者。
- **時間**：代理人應從過去學習，在當下提供相關提示，並為未來調整。
- **核心**：接納不確定性，但透過透明性與使用者控制建立信任。

## 5. 主要課程重點摘要

- **Lesson 1**：代理人是系統，而不僅僅是模型。它們會感知、推理並行動。
- **Lesson 2**：Microsoft Agent Framework 抽象化了工具呼叫與狀態管理的複雜性。
- **Lesson 3**：設計時應考量透明性與使用者控制。
- **Lesson 4**：工具是代理人的「雙手」。schema 定義對 LLM 能理解如何使用工具至關重要。
- **Lesson 7**：規劃是代理人的「執行功能」，使其能處理複雜的工作流程。
