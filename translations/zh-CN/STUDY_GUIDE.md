# 初学者的 AI 代理 - 学习指南与课程总结

本指南总结了“初学者的 AI 代理”课程，并解释了构建 AI 代理的关键概念、框架和设计模式。

## 1. AI 代理简介

**什么是 AI 代理？**  
AI 代理是通过赋予大型语言模型（LLM）**工具**、**知识**和**记忆**的访问能力，从而扩展其功能的系统。与仅基于训练数据生成文本的标准 LLM 聊天机器人不同，AI 代理可以：  
- **感知**其环境（通过传感器或输入）。  
- **推理**解决问题的方法。  
- **行动**以改变环境（通过执行器或工具执行）。  

**代理的关键组成部分：**  
- **环境**：代理运行的空间（例如，预订系统）。  
- **传感器**：收集信息的机制（例如，读取 API）。  
- **执行器**：执行动作的机制（例如，发送电子邮件）。  
- **大脑（LLM）**：规划和决定采取何种行动的推理引擎。

## 3. 代理设计模式

设计模式有助于构建结构化的代理操作，可靠地解决问题。

### **工具使用模式**（第4课）  
此模式使代理能够与外部世界交互。  
- **概念**：代理被提供一个“schema”（可用函数及其参数列表）。LLM 根据用户请求决定调用*哪个*工具及*什么*参数。  
- **流程**：用户请求 -> LLM -> **工具选择** -> **工具执行** -> LLM（附带工具输出）-> 最终响应。  
- **使用场景**：获取实时数据（天气、股票价格）、执行计算、运行代码。  

### **规划模式**（第7课）  
此模式使代理能够解决复杂的多步骤任务。  
- **概念**：代理将高层目标拆解成一系列更小的子任务。  
- **方法**：  
  - **任务分解**：将“计划旅行”拆成“订机票”、“订酒店”、“租车”。  
  - **迭代规划**：根据前一步骤结果重新评估计划（如航班满员，则选择不同日期）。  
- **实现**：通常涉及生成结构化计划（如 JSON）的“规划者”代理，供其他代理执行。  

## 4. 设计原则

设计代理时，应考虑三个维度：  
- **空间**：代理应连接人和知识，易于访问但不干扰。  
- **时间**：代理应从*过去*学习，在*现在*提供相关提示，并适应*未来*。  
- **核心**：接受不确定性，但通过透明性和用户控制建立信任。  

## 5. 关键课程总结

- **第1课**：代理是系统，不只是模型。它们能感知、推理和行动。  
- **第2课**：Microsoft Agent Framework 抽象了工具调用和状态管理的复杂性。  
- **第3课**：设计时应考虑透明性和用户控制。  
- **第4课**：工具是代理的“手”。schema 定义对 LLM 理解工具使用至关重要。  
- **第7课**：规划是代理的“执行功能”，使其能处理复杂工作流。
