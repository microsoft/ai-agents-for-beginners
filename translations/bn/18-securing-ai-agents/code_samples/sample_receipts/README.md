# নমুনা রসিদ ফিক্সচারগুলি

বিনা নোটবুক চালানোর জন্য তিনটি পূর্ব-তৈরি রসিদ ফাইল পরিদর্শনের জন্য।

| ফাইল | এটি কি |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` টুল কলের জন্য একটি বৈধ স্বাক্ষরিত রসিদ। যাচাইকরণ সত্য (True) রিটার্ন করে। |
| `02_tampered_receipt.json` | স্বাক্ষরের পর একটি ক্ষেত্র পরিবর্তিত একই রসিদ। যাচাইকরণ মিথ্যা (False) রিটার্ন করে। |
| `03_chain_three_receipts.json` | তিনটি বৈধ রসিদের (search, hold, book) একটি চেইন যেখানে `previous_receipt_hash` প্রতিটির পূর্ববর্তী রসিদের সাথে সংযুক্ত করে। |

## নমুনাগুলি যাচাই করা

নোটবুক চারটি বিভাগে যাচাই আলোচনা করে। নোটবুক বর্ণনা চালানো ছাড়াই সরাসরি এই ফিক্সচারের যাচাই করতে:

```python
import json
from pathlib import Path

# ধরে নিচ্ছে আপনি ইম্পোর্ট এবং সাহায্যকারী ফাংশনগুলি সম্পন্ন করেছেন
# 18-signed-receipts.ipynb এর ১ এবং ২ নম্বর সেকশন থেকে।

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # সত্য

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # মিথ্যা

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

বাইট পুনঃউৎপাদনের জন্য স্থির সময়মুদ্রা সহ। পুনঃতৈরির জন্য:

## কাঁচা JSON পরিদর্শন থেকে শিক্ষার্থীরা কী শেখে

কাঁচা রসিদ ফরম্যাট পড়া একটি অন্তর্দৃষ্টি গড়ে তুলে যা নোটবুকের সেলগুলো সবসময় দেয় না। JSON দ্রুত দেখা শিক্ষার্থীরা প্রায়শই লক্ষ্য করে:

1. স্বাক্ষর একটি অস্পষ্ট base64url স্ট্রিং, কিন্তু প্রতিটি অন্য ক্ষেত্র সোজা পাঠযোগ্য JSON। স্বাক্ষর বিষয়বস্তু এনক্রিপ্ট করে না; এটি বিষয়বস্তু নিশ্চিত করে।
2. `public_key` রসিদে এম্বেড করা আছে। একজন নিরীক্ষক যাচাইয়ের জন্য অন্য কিছুই প্রয়োজন না
   (বিশ্বাস করার শর্তে যে চাবিটি সত্যিই দাবিকৃত ইস্যুকর্তার; পরিচয় অবকাঠামো পাঠের README দেখুন)।
3. কোনো ক্ষেত্রের একটি অক্ষর পরিবর্তন করা, তারপর এই ফাইলটি `02_tampered_receipt.json` এর সাথে পুনরায় তুলনা করা হলে, বাইট-স্তরের প্রক্রিয়া স্পষ্ট হয়।
