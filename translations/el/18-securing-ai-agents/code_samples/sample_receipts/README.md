# Προκαθορισμένα Αρχεία Απόδειξης

Τρία προ-δημιουργημένα αρχεία απόδειξης για επιθεώρηση χωρίς να τρέξετε το σημειωματάριο.

| Αρχείο | Τι είναι |
|---|---|
| `01_valid_receipt.json` | Έγκυρη υπογεγραμμένη απόδειξη για κλήση εργαλείου `lookup_flights`. Η επαλήθευση επιστρέφει True. |
| `02_tampered_receipt.json` | Η ίδια απόδειξη με ένα πεδίο τροποποιημένο μετά την υπογραφή. Η επαλήθευση επιστρέφει False. |
| `03_chain_three_receipts.json` | Μια αλυσίδα τριών έγκυρων αποδείξεων (αναζήτηση, κράτηση, επιβεβαίωση) με `previous_receipt_hash` που συνδέει κάθε απόδειξη με την προηγούμενη. |

## Επαλήθευση των δειγμάτων

Το σημειωματάριο διαπερνά την επαλήθευση σε τέσσερις ενότητες. Για να επαληθεύσετε αυτά τα προκαθορισμένα αρχεία
άμεσα χωρίς να περάσετε από το αφήγημα του σημειωματαρίου:

```python
import json
from pathlib import Path

# Υποθέτει ότι έχετε ολοκληρώσει τις εισαγωγές και τις βοηθητικές συναρτήσεις
# από τα τμήματα 1 και 2 του 18-signed-receipts.ipynb.

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # Αληθές

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # Ψευδές

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## Πώς δημιουργήθηκαν αυτά

Τα προκαθορισμένα αρχεία χρησιμοποιούν την ίδια διαδρομή κώδικα με το σημειωματάριο, με ένα σταθερό κλειδί υπογραφής
και σταθερές χρονικές σφραγίδες για αναπαραγωγή byte. Για να δημιουργήσετε ξανά:

```bash
python3 generate_fixtures.py
```

(Το script βρίσκεται στο `generate_fixtures.py` σε αυτόν τον φάκελο.)

## Τι μαθαίνουν οι φοιτητές εξετάζοντας το ακατέργαστο JSON

Η ανάγνωση της ακατέργαστης μορφής απόδειξης χτίζει την αντίληψη που τα κελιά στο σημειωματάριο
δεν παρέχουν πάντα. Οι φοιτητές που ρίχνουν μια ματιά στο JSON συχνά παρατηρούν:

1. Η υπογραφή είναι μια αδιαφανής συμβολοσειρά base64url, αλλά κάθε άλλο πεδίο είναι απλό
   αναγνώσιμο JSON. Η υπογραφή δεν κρυπτογραφεί το περιεχόμενο· το επιβεβαιώνει.
2. Το `public_key` είναι ενσωματωμένο στην απόδειξη. Ένας ελεγκτής δεν χρειάζεται τίποτε άλλο
   για να επαληθεύσει (υπό την προϋπόθεση ότι εμπιστεύεται ότι το κλειδί ανήκει όντως στον αιτούμενο
   εκδότη· δείτε το README του μαθήματος για την υποδομή ταυτότητας).
3. Η τροποποίηση ενός μόνο χαρακτήρα οποιουδήποτε πεδίου, και μετά η επανάληψη της σύγκρισης αυτού του αρχείου με το
   `02_tampered_receipt.json`, καταστούν το μηχανισμό σε επίπεδο byte πιο συγκεκριμένο.
