# Πράκτορες AI για Αρχάριους - Οδηγός Μελέτης & Περίληψη Μαθήματος

Αυτός ο οδηγός παρέχει μια περίληψη του μαθήματος "Πράκτορες AI για Αρχάριους" και εξηγεί βασικές έννοιες, πλαίσια και μοτίβα σχεδίασης για την κατασκευή Πρακτόρων AI.

## 1. Εισαγωγή στους Πράκτορες AI

**Τι είναι οι Πράκτορες AI;**  
Οι Πράκτορες AI είναι συστήματα που επεκτείνουν τις δυνατότητες των Μεγάλων Γλωσσικών Μοντέλων (LLMs) δίνοντάς τους πρόσβαση σε **εργαλεία**, **γνώση** και **μνήμη**. Σε αντίθεση με έναν τυπικό chatbot LLM που παράγει μόνο κείμενο βάσει των δεδομένων εκπαίδευσης, ένας Πράκτορας AI μπορεί:  
- **Να αντιλαμβάνεται** το περιβάλλον του (μέσω αισθητήρων ή εισροών).  
- **Να συλλογίζεται** πώς να λύσει ένα πρόβλημα.  
- **Να δρουν** για να αλλάξουν το περιβάλλον (μέσω ενεργοποιητών ή εκτέλεσης εργαλείων).

**Βασικά Στοιχεία ενός Πράκτορα:**  
- **Περιβάλλον**: Ο χώρος στον οποίο λειτουργεί ο πράκτορας (π.χ., ένα σύστημα κρατήσεων).  
- **Αισθητήρες**: Μηχανισμοί συλλογής πληροφοριών (π.χ., ανάγνωση API).  
- **Ενεργοποιητές**: Μηχανισμοί εκτέλεσης ενεργειών (π.χ., αποστολή email).  
- **Εγκέφαλος (LLM)**: Η μονάδα συλλογιστικής που σχεδιάζει και αποφασίζει ποιες ενέργειες θα εκτελεστούν.

## 2. Πλαίσια Πρακτόρων

Το μάθημα χρησιμοποιεί **Microsoft Agent Framework (MAF)** με **Azure AI Foundry Agent Service V2** για την κατασκευή πρακτόρων:

| Στοιχείο | Εστίαση | Κατάλληλο Για |
|-----------|----------|---------------|
| **Microsoft Agent Framework** | Ενιαίο SDK σε Python/C# για πράκτορες, εργαλεία και ροές εργασίας | Κατασκευή πρακτόρων με εργαλεία, ροές εργασίας πολλών πρακτόρων, και προτύπα παραγωγής. |  
| **Azure AI Foundry Agent Service** | Διαχειριζόμενο περιβάλλον cloud εκτέλεσης | Ασφαλής, επεκτάσιμη ανάπτυξη με ενσωματωμένη διαχείριση κατάστασης, παρατηρησιμότητα και εμπιστοσύνη. |

## 3. Μοτίβα Σχεδίασης Πρακτόρων

Τα μοτίβα σχεδίασης βοηθούν στη δομή του τρόπου λειτουργίας των πρακτόρων για αξιόπιστη επίλυση προβλημάτων.

### **Μοτίβο Χρήσης Εργαλείου** (Μάθημα 4)  
Αυτό το μοτίβο επιτρέπει στους πράκτορες να αλληλεπιδρούν με τον εξωτερικό κόσμο.  
- **Έννοια**: Ο πράκτορας παρέχεται με ένα "σχήμα" (λίστα διαθέσιμων λειτουργιών και των παραμέτρων τους). Το LLM αποφασίζει *ποιο* εργαλείο θα καλέσει και με *ποια* επιχειρήματα βάσει του αιτήματος χρήστη.  
- **Ροή**: Αίτημα Χρήστη -> LLM -> **Επιλογή Εργαλείου** -> **Εκτέλεση Εργαλείου** -> LLM (με έξοδο εργαλείου) -> Τελική Απόκριση.  
- **Περίπτωσεις Χρήσης**: Ανάκτηση δεδομένων σε πραγματικό χρόνο (καιρός, τιμές μετοχών), εκτέλεση υπολογισμών, εκτέλεση κώδικα.

### **Μοτίβο Σχεδιασμού** (Μάθημα 7)  
Αυτό το μοτίβο επιτρέπει στους πράκτορες να επιλύουν σύνθετα, πολύ-βηματικά καθήκοντα.  
- **Έννοια**: Ο πράκτορας διασπά ένα υψηλού επιπέδου στόχο σε μια ακολουθία μικρότερων υποκαθηκόντων.  
- **Προσεγγίσεις**:  
  - **Ανάλυση Καθηκόντων**: Διαχωρισμός του "Σχεδίασε ένα ταξίδι" σε "Κράτηση πτήσης", "Κράτηση ξενοδοχείου", "Ενοικίαση αυτοκινήτου".  
  - **Επαναληπτικός Σχεδιασμός**: Επαναξιολόγηση του σχεδίου βάσει των αποτελεσμάτων προηγούμενων βημάτων (π.χ. αν η πτήση είναι γεμάτη, επιλέξτε άλλη ημερομηνία).  
- **Υλοποίηση**: Συχνά περιλαμβάνει έναν πράκτορα "Σχεδιαστή" που δημιουργεί ένα δομημένο σχέδιο (π.χ., JSON) το οποίο εκτελείται από άλλους πράκτορες.

## 4. Αρχές Σχεδίασης

Κατά το σχεδιασμό πρακτόρων, λάβετε υπόψη τρεις διαστάσεις:  
- **Χώρος**: Οι πράκτορες πρέπει να συνδέουν ανθρώπους και γνώση, να είναι προσβάσιμοι αλλά διακριτικοί.  
- **Χρόνος**: Οι πράκτορες πρέπει να μαθαίνουν από το *Παρελθόν*, να παρέχουν σχετικές υποδείξεις στο *Παρόν* και να προσαρμόζονται για το *Μέλλον*.  
- **Κέντρο**: Αποδεχτείτε την αβεβαιότητα, αλλά εδραιώστε εμπιστοσύνη μέσω διαφάνειας και ελέγχου από τον χρήστη.

## 5. Περίληψη Βασικών Μαθημάτων

- **Μάθημα 1**: Οι πράκτορες είναι συστήματα, όχι απλά μοντέλα. Αντιλαμβάνονται, συλλογίζονται και δρουν.  
- **Μάθημα 2**: Το Microsoft Agent Framework αφαιρεί την πολυπλοκότητα της κλήσης εργαλείων και της διαχείρισης κατάστασης.  
- **Μάθημα 3**: Σχεδιάστε με διαφάνεια και έλεγχο από τον χρήστη στο μυαλό.  
- **Μάθημα 4**: Τα εργαλεία είναι τα "χέρια" του πράκτορα. Ο ορισμός του σχήματος είναι κρίσιμος για το LLM να κατανοήσει πώς να τα χρησιμοποιήσει.  
- **Μάθημα 7**: Ο σχεδιασμός είναι η "εκτελεστική λειτουργία" του πράκτορα, επιτρέποντας του να διαχειριστεί σύνθετες ροές εργασίας.
