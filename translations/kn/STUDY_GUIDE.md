# ಆರಂಭಿಕರಿಗಾಗಿ AI ಏಜೆಂಟ್ಸ್ - ಅಧ್ಯಯನ ಮಾರ್ಗದರ್ಶಿ ಮತ್ತು ಕೋರ್ಸ್ ಸಾರಾಂಶ

ಈ ಮಾರ್ಗದರ್ಶಿ "ಆರ್‌ಟಿಫಿಶಿಯಲ್ ಇಂಟಲಿಜೆನ್ಸ್ ಏಜೆಂಟ್ಸ್ ಫಾರ್ ಬೆಗಿನರ್ಸ್" ಕೋರ್ಸ್‌ನ ಸಾರಾಂಶವನ್ನು ಒದಗಿಸುತ್ತದೆ ಮತ್ತು AI ಏಜೆಂಟ್ಸ್ ನಿರ್ಮಿಸಲು ключ್ ಅಂಶಗಳು, ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು ಮತ್ತು ವಿನ್ಯಾಸ ಮಾದರಿಗಳನ್ನು ವಿವರಿಸುತ್ತದೆ.

## 1. AI ಏಜೆಂಟ್ಸ್ ಗೆ ಪರಿಚಯ

**AI ಏಜೆಂಟ್ಸ್ ಎಂದರೆ ಏನು?**  
AI ಏಜೆಂಟ್ಸ್ ಎಂಬವು ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾದರಿಗಳ (LLMs) ಸಾಮರ್ಥ್ಯಗಳನ್ನು ವಿಸ್ತರಿಸುವ системಗಳು, ಅವುಗಳಿಗೆ **ಟೂಲ್‌ಗಳು**, **ಜ್ಞಾನ**, ಮತ್ತು **ಸ್ಮೃತಿ** ಅನ್ನು ಪ್ರವೇಶ ನೀಡುತ್ತವೆ. ತರಬೇತಿ ಡೇಟಾ ಆಧರಿಸಿ ಪಠ್ಯವನ್ನು ಮಾತ್ರ ರಚಿಸುವ ಸಾಮಾನ್ಯ LLM ಚಾಟ್‌ಬಾಟ್‌ನಂತೆ ಅಲ್ಲ, AI ಏಜೆಂಟ್ ಕೆಳಗಿನ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಬಹುದು:  
- **ಮಾತುತುದಿರು** ತನ್ನ ಪರಿಸರವನ್ನು (ಸೆನ್ಸಾರುಗಳು ಅಥವಾ ಇನ್‌ಪುಟ್‌ಗಳ ಮೂಲಕ).  
- **ತರ್ಕ ಮಾಡು** ಸಮಸ್ಯೆಯನ್ನು ಹೇಗೆ ಪರಿಹರಿಸುವುದು ಎಂಬುದನ್ನು.  
- **ಲೈಪು ಮಾಡುವಿಕೆ ಮಾಡು** ಪರಿಸರವನ್ನು ಬದಲಾಯಿಸಲು (ಅಕ್ಟ್ಯುಯೇಟರ್ಸ್ ಅಥವಾ ಟೂಲ್ ನಿರ್ವಹಣೆ ಮೂಲಕ).

**ಏಜೆಂಟ್‌ನ ಪ್ರಮುಖ ಅಂಶಗಳು:**  
- **ಪರಿಸರ**: ಏಜೆಂಟ್ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಸ್ಥಳ (ಉದಾ: ಬುಕ್ಕಿಂಗ್ ಸಿಸ್ಟಮ್).  
- **ಸೆನ್ಸಾರುಗಳು**: ಮಾಹಿತಿಯನ್ನು ಸಂಗ್ರಹಿಸುವ ಯಂತ್ರಗಳು (ಉದಾ: API ಓದುವುದು).  
- **ಅಕ್ಟ್ಯುಯೇಟರ್ಸ್**: ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಯಂತ್ರಗಳು (ಉದಾ: ಇಮೇಲ್ ಕಳುಹಿಸುವುದು).  
- **ಬ್ರೈನ್ (LLM)**: ಕಾರ್ಯಗಳನ್ನು ಯೋಜಿಸುವ ಮತ್ತು ನಿರ್ಧರಿಸುವ ಯಂತ್ರ.

## 2. ಏಜೆಂಟಿಕ್ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು

ಕೋರ್ಸ್ **Microsoft Agent Framework (MAF)** ಮತ್ತು **Azure AI Foundry Agent Service V2** ಅನ್ನು ಏಜೆಂಟ್‌ಗಳ ನಿರ್ಮಾಣಕ್ಕಾಗಿ ಬಳಕೆ ಮಾಡುತ್ತದೆ:

| ಅಂಶ            | ಫೋಕಸ್      | ಅತ್ಯುತ್ತಮ ಬಳಕೆಗೆ            |
|-----------------|------------|----------------------------|
| **Microsoft Agent Framework**  | ಏಜೆಂಟ್‌ಗಳು, ಟೂಲ್‌ಗಳು, ಮತ್ತು ವರ್ಕ್ಫ್ಲೋಗಳಿಗಾಗಿ ಏಕೀಕೃತ Python/C# SDK | ಟೂಲ್‌ಗಳೊಂದಿಗೆ ಏಜೆಂಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು, ಬಹು-ಏಜೆಂಟ್ ವರ್ಕ್ಫ್ಲೋಗಳು, ಮತ್ತು ಉತ್ಪಾದನಾ ಮಾದರಿಗಳು. |
| **Azure AI Foundry Agent Service** | ನಿರ್ವಹಿಸಲ್ಪಡುವ ಕ್ಲೌಡ್ ಕಾರ್ಯ ನಿರ್ವಹಣೆ | ಒಳಗೊಂಡ ಸ್ಥಿತಿ ನಿರ್ವಹಣೆ, ವೀಕ್ಷಣೆ ಮತ್ತು ನಂಬಿಕೆಯಿಂದ ಸುರಕ್ಷಿತ, ವ್ಯಾಪಕ ನಿಯೋಜನೆ. |

## 3. ಏಜೆಂಟಿಕ್ ವಿನ್ಯಾಸ ಮಾದರಿಗಳು

ವಿನ್ಯಾಸ ಮಾದರಿಗಳು ಏಜೆಂಟ್‌ಗಳು ಸಮಸ್ಯೆಗಳನ್ನು ನಂಬಿಕೆ ಹಾಕಿಕೊಳ್ಳುವಂತೆ ಪರಿಹರಿಸಲು ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸಬೇಕು ಎಂಬುದನ್ನು ರಚಿಸಲು ಸಹಾಯಕ.

### **ಟೂಲ್ ಉಪಯೋಗ ಮಾದರಿ** (ಪಾಠ 4)  
ಈ ಮಾದರಿ ಏಜೆಂಟ್‌ಗಳಿಗೆ ಹೊರಗಿನ ಜಗತ್ತಿನೊಂದಿಗೆ ಸಂವಹನ ಮಾಡಲು ಸಹಾಯಕ.  
- **ಧಾರಣೆ**: ಏಜೆಂಟ್‌ಗೆ "ಸ್ಕೀಮಾ" (ಲಭ್ಯವಿರುವ ಕಾರ್ಯಚಟುವಟಿಕೆಗಳ ಮತ್ತು ಅವುಗಳ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ಪಟ್ಟಿ) ಒದಗಿಸಲಾಗುತ್ತದೆ. LLM ಯು ಬಳಕೆದಾರನ ವಿನಂತಿಯ ಆಧಾರದಲ್ಲಿ *ಯಾವ* ಟೂಲ್ ಅನ್ನು ಕರೆಸಬೇಕೆಂದು ಮತ್ತು *ಯಾವ* ಆರ್ಗ್ಯೂಮೆಂಟ್‌ಗಳೊಂದಿಗೆ ಎಂದು ನಿರ್ಧರಿಸುತ್ತದೆ.  
- **ಹಣೆಮಾರ್ಗ**: ಬಳಕೆದಾರ ವಿನಂತಿ -> LLM -> **ಟೂಲ್ ಆಯ್ಕೆ** -> **ಟೂಲ್ ನಿರ್ವಹಣೆ** -> LLM (ಟೂಲ್ ಔಟ್‌ಪುಟ್‌ನೊಂದಿಗೆ) -> ಅಂತಿಮ ಉತ್ತರ.  
- **ಬಳಕೆ ಪ್ರಕರಣಗಳು**: ನಿಜ ಕಾಲದ ಡೇಟಾವನ್ನು ಪಡೆದುಕೊಳ್ಳುವುದು (ಹವಾಮಾನ, ಸ್ಟಾಕ್ ಬೆಲೆಗಳು), ಗಣನೆಗಳು ನಡೆಸುವುದು, ಕೋಡ್ ನಿರ್ವಹಣೆ.

### **ಯೋಜನೆ ಮಾದರಿ** (ಪಾಠ 7)  
ಈ ಮಾದರಿ ಏಜೆಂಟ್‌ಗಳಿಗೆ ಸಂಕೀರ್ಣ, ಬಹು ಹಂತದ ಕೆಲಸಗಳನ್ನು ಪರಿಹರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.  
- **ಧಾರಣೆ**: ಏಜೆಂಟ್ ಒಂದು ಉನ್ನತ ಮಟ್ಟದ ಗುರಿಯನ್ನು ಹಿಂದುಳಿದ ಕಡಿಮೆ ಹಂತದ ಉಪಕಾರ್ಯಗಳ ಸರಣಿಗಾಗಿ ವಿಂಗಡಿಸುತ್ತದೆ.  
- **ಮನೋಭಾವಗಳು**:  
  - **ಕಾರ್ಯ ವಿಭಜನೆ**: "ಪ್ರಯಾಣ ಯೋಜನೆ" ಅನ್ನು "ವಿಮಾನ ಬುಕ್ಕಿಂಗ್", "ಹೋಟೆಲ್ ಬುಕ್ಕಿಂಗ್", "ಕಾರು ಬಾಡಿಗೆ" ಎಂದು ವಿಭಾಗಿಸುವುದು.  
  - **ಪುನರ್ ವಿಮರ್ಶೆ ಯೋಜನೆ**: ಹಿಂದಿನ ಹಂತಗಳ ಔಟ್‌ಪುಟ್ ಆಧಾರದಲ್ಲಿಯೇ ಯೋಜನೆಯನ್ನು ಮರು ಮೌಲ್ಯಮಾಪನ ಮಾಡುವುದು (ಉದಾ: ವಿಮಾನ ತುಂಬಿದ್ದರೆ ಬೇರೆ ದಿನಾಂಕ ಆಯ್ಕೆಮಾಡುವುದು).  
- **ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆ**: ಬಹುತೇಕ "ಪ್ಲ್ಯಾನರ್" ಏಜೆಂಟ್ ಒಬ್ಬನಾಗಿದ್ದು, ರಚನಾತ್ಮಕ ಯೋಜನೆಯನ್ನು (ಉದಾ: JSON) ಸೃಷ್ಟಿಸುತ್ತದೆ ಮತ್ತು ನಂತರ ಇತರೆ ಏಜೆಂಟ್‌ಗಳು ಅದನ್ನು ನಿರ್ವಹಿಸುತ್ತವೆ.

## 4. ವಿನ್ಯಾಸ ತತ್ವಗಳು

ಏಜೆಂಟ್‌ಗಳನ್ನು ವಿನ್ಯಾಸ ಮಾಡಬೇಕಾಗಿರುವಾಗ ಮೂರು ಆಯಾಮಗಳನ್ನು ಪರಿಗಣಿಸಿ:  
- **ಅವರಣ**: ಏಜೆಂಟ್‌ಗಳು ಜನರನ್ನು ಮತ್ತು ಜ್ಞಾನವನ್ನು ಸಂಪರ್ಕಿಸಬೇಕು, ಸುಲಭವಾಗಿ ಲಭ್ಯವಾಗಿದ್ದು ಅತಿವಾಹಕವಾಗಿರಬಾರದು.  
- **ಸಮಯ**: ಏಜೆಂಟ್‌ಗಳು *ಹಿಂದಿನ* ಅನುಭವದಿಂದ ಕಲಿತುಕೊಳ್ಳಬೇಕು, *ಈ ಕ್ಷಣ* ಸಂಬಂಧಿಸಿದ ಸೂಚನೆಗಳನ್ನು ನೀಡಬೇಕು ಮತ್ತು *ಭವಿಷ್ಯ*ಗಾಗಿ ಹೊಂದಿಕೊಳ್ಳಬೇಕು.  
- **ಮೂಲ**: ಅನಿಶ್ಚಿತತೆಯನ್ನು ಸ್ವೀಕರಿಸಿ, ಆದರೆ ಪಾರದರ್ಶಕತೆ ಮತ್ತು ಬಳಕೆದಾರ ನಿಯಂತ್ರಣದಿಂದ ನಂಬಿಕೆಯನ್ನು ಖಡ್ಗಗೊಳಿಸಬೇಕು.

## 5. ಪ್ರಮುಖ ಪಾಠಗಳ ಸಾರಾಂಶ

- **ಪಾಠ 1**: ಏಜೆಂಟ್‌ಗಳು ವ್ಯವಸ್ಥೆಗಳು, ಮಾದರಿಗಳಲ್ಲ. ಅವು ಮಾತುತುದಿರು, ತರ್ಕ ಮಾಡು ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಿಸು.  
- **ಪಾಠ 2**: Microsoft Agent Framework ಟೂಲ್ ಕರೆಯುವಿಕೆ ಮತ್ತು ಸ್ಥಿತಿ ನಿರ್ವಹಣೆಯ ಸಂಕೀರ್ಣತೆಯನ್ನು ಸರಳಗೊಳಿಸುತ್ತದೆ.  
- **ಪಾಠ 3**: ಪಾರದರ್ಶಕತೆ ಮತ್ತು ಬಳಕೆದಾರ ನಿಯಂತ್ರಣವನ್ನು ಮನಸ್ಸಿನಲ್ಲಿ ಇಟ್ಟುಕೊಂಡು ವಿನ್ಯಾಸ ಮಾಡು.  
- **ಪಾಠ 4**: ಟೂಲ್‌ಗಳು ಏಜೆಂಟ್‌ನ "ಹಸ್ತಗಳು". ಸ್ಕೀಮಾ ವ್ಯಾಖ್ಯಾನ LLM ಗೆ ಅವುಗಳನ್ನು ಹೇಗೆ ಉಪಯೋಗಿಸಬೇಕು ಎಂಬುದನ್ನು ಅನುವಾದಿಸಲು ಅಗತ್ಯ.  
- **ಪಾಠ 7**: ಯೋಜನೆ ಏಜೆಂಟ್‌ನ "ನಿರ್ವಹಣಾ ಕಾರ್ಯ", ಅದು ಅದನ್ನು ಸಂಕೀರ್ಣ ಕೆಲಸವಾಹಿಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.
