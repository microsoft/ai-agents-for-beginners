# ಮಾದರಿ ರಸೀದಿ ಫಿಕ್ಸ್ಚರ್ಸ್

ನೋಟ್ಬುಕ್ ಅನ್ನು ಚಾಲನೆಯಲ್ಲಿಡದೇ ಪರಿಶೀಲನೆಗಾಗಿ ಮೂರು ಪೂರ್ವ ಉತ್ಪಾದಿತ ರಸೀದಿ ಕಡತಗಳು.

| ಫೈಲ್ | ಅದು ಏನು |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` ಸಾಧನ ಕರೆಗಾಗಿ ಮಾನ್ಯವಾದ ಸಹಿ ಪಡೆದ ರಸೀದಿ. ಪರಿಶೀಲನೆ ಸತ್ಯವನ್ನು ನೀಡುತ್ತದೆ. |
| `02_tampered_receipt.json` | ಸಹಿ ಸೂಚಿಸುವ ನಂತರ ಒಂದು ಕ್ಷೇತ್ರವನ್ನು ಬದಲಿಸಿದ ಅದೇ ರಸೀದಿ. ಪರಿಶೀಲನೆ ಸುಳ್ಳನ್ನು ನೀಡುತ್ತದೆ. |
| `03_chain_three_receipts.json` | ಮೂರು ಮಾನ್ಯ ರಸೀದಿಗಳ ಸರಪಳಿ (ಹುಡುಕು, ಹೂಡಿಕೆ, ಬುಕ್) `previous_receipt_hash` ಮೂಲಕ ಪ್ರತಿ ರಸೀದಿಯನ್ನು ಹಿಂದಿನುದಕ್ಕೆ ಲಿಂಕ್ ಮಾಡಿದ್ದು. |

## ಮಾದರಿಗಳನ್ನು ಪರಿಶೀಲಿಸುವುದು

ನೋಟ್ಬುಕ್ ನಾಲ್ಕು ವಿಭಾಗಗಳಲ್ಲಿ ಪರಿಶೀಲನೆಯನ್ನು ವಿವರಿಸುತ್ತದೆ. ಈ ಫಿಕ್ಸ್ಚರ್ಗಳನ್ನು ನೋಟ್ಬುಕ್ ಹಾಡು ಹೇಳಿಕೆಯಿಲ್ಲದೆ ನೇರವಾಗಿ ಪರಿಶೀಲಿಸಲು:

```python
import json
from pathlib import Path

# ನೀವು ಆಮದುಗಳು ಮತ್ತು ಸಹಾಯಕ ಫಂಕ್ಷನ್‌ಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ್ದೀರಿ ಎಂದು فرضಿಸುತ್ತದೆ
# 18-signed-receipts.ipynb ನ 1 ಮತ್ತು 2 ಭಾಗಗಳಿಂದ.

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # ನಿಜ

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # ತಪ್ಪು

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## ಇವು ಹೇಗೆ ಉತ್ಪಾದಿಸಲ್ಪಟ್ಟವು

ನೋಟ್ಬುಕ್‌ನ ಅನುಸರಣೆ ಜತೆಗೆ ಇವುಗಳಿಗೂ ಒಂದೇ ಸಂಕೇತಿಸುವ ಕೀ ಮತ್ತು ಬೈಟ್ ಪುನಃ ಉತ್ಪಾದನೀಯತೆಯಿಗಾಗಿ ಸ್ಥಿರವಾಗಿರುವ ಟೈಮ್ಸ್‍ಟ್ಯಾಂಪ್ ಬಳಸಲಾಗಿದೆ. ಪುನಃ ಉತ್ಪಾದಿಸಲು:

```bash
python3 generate_fixtures.py
```

## ವಿದ್ಯಾರ್ಥಿಗಳು ಕಚಗುಳಿಯ JSON ಪರಿಶೀಲನೆಯಿಂದ ಏನು ಕಲಿಯುತ್ತಾರೆ

ಕಚಗುಳಿಯ ರಸೀದಿ ಸ್ವರೂಪ ಓದುವಿಕೆ ನೋಟ್ಬುಕ್‌ನ ಸೆಲ್‍ಗಳು ಪ್ರತಿದಿನ ನೀಡುವಂತೆ ಇಲ್ಲದ ಸಂವೇದನೆಯನ್ನು ನಿರ್ಮಿಸುತ್ತದೆ. JSON ತಿರುವು ಮಾಡುವ ವಿದ್ಯಾರ್ಥಿಗಳು ಸಾಮಾನ್ಯವಾಗಿ ಗಮನಿಸುವುದು:

1. ಸಹಿ ಒಂದು ಅಸ್ಪಷ್ಟವಾದ base64url ಸರಪಳಿಯಾಗಿದ್ದು, ಇತರೆ ಎಲ್ಲ ಕ್ಷೇತ್ರಗಳು ಸರಳ ಓದಬಹುದಾದ JSON ಆಗಿವೆ. ಸಹಿ ವಿಷಯವನ್ನು ಗೂಢಲಿಪಿಯಾಗಿಸುವುದಿಲ್ಲ; ಅದನ್ನು ದೃಢಪಟ್ಟಿಸುತ್ತದೆ.
2. `public_key` ರಸೀದಿಯಲ್ಲಿ ಆಳವಾಗಿದೆ. ಪರಿಶೀಲಿಸಲು ಅಕೌಂಟರ್‌ಗೆ ಇನ್ನಾವುದು ಬೇಕಾಗಿಲ್ಲ (ಅದ್ರೆ ಕೀ ನಿಜವಾಗಿಯೂ ತಿಳಿಸಿದ್ದಾರೆ ಎಂಬ ನಂಬಿಕೆ ಇದ್ದರೆ; ಗುರುತು ಮೂಲಸೌಕರ್ಯ ಕುರಿತ ಪಾಠ README ನೋಡಿ).
3. ಯಾವ ಕ್ಷೇತ್ರದ ಯಾವುದೇ ಅಕ್ಷರವನ್ನು ಬದಲಾಯಿಸಿ ಈ ಕಡತವನ್ನು `02_tampered_receipt.json` ಜೊತೆಗೆ ಮತ್ತೆ ಹೋಲಿಸಿದಾಗ, ಬೈಟ್ ಮಟ್ಟದ ವ್ಯವಸ್ಥೆ ಸ್ಪಷ್ಟವಾಗುತ್ತದೆ.
