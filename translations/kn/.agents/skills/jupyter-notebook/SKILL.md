# Jupyter Notebook ಕೌಶಲ್ಯ

ಸ್ವಚ್ಚ, ಪುನರಾವರ್ತನೀಯ Jupyter ನೋಟ್ಬುಕ್‌ಗಳನ್ನು ಎರಡು ಪ್ರಮುಖ ಮೋಡ್‌ಗಳಿಗಾಗಿ ರಚಿಸಿ:

- ಪ್ರಯೋಗಗಳು ಮತ್ತು ಅನ್ವೇಷಣಾತ್ಮಕ ವಿಶ್ಲೇಷಣೆ
- ಟ್ಯೂಟೋರಿಯಲ್ಗಳು ಮತ್ತು ಕಲಿಕೆಯ-ಕೇಂದ್ರೀಕೃತ ವಾಕ್‌ಥ್ರೂಗಳು

ಸಂರಚನೆ ಸ್ಥಿರವಾಗಿರಲು ಮತ್ತು ಕಡಿಮೆ JSON ತಪ್ಪುಗಳಿಗಾಗಿ ಪ್ಯಾಕ್ಡ್ ಟೆಂಪ್ಲೇಟ್ಗಳು ಮತ್ತು ಸಹಾಯಕರ ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಪ್ರಾಧಾನ್ಯ ನೀಡಿ.

## ಯಾವಾಗ ಬಳಸಬೇಕು
- ಹೊಸ `.ipynb` ನೋಟ್ಬುಕ್ ಅನ್ನು ಶೂನ್ಯದಿಂದ ರಚಿಸಿ.
- ಉಹಾಪೋಹದ ಟಿಪ್ಪಣಿಗಳು ಅಥವಾ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ರಚನಾತ್ಮಕ ನೋಟ್ಬುಕ್‌ಗಾಗಿಯೇ ಪರಿವರ್ತಿಸು.
- നിലവ arro existನೋಟ್ ಬುಕ್ ಅನ್ನು ಪುನರ್‍ರೂಪಗೊಳಿಸಿ ώστε ಹೆಚ್ಚು ಪುನರಾವರ್ತನೀಯ ಮತ್ತು ಓದಲು ಸುಲಭವಾಗುವಂತೆ ಮಾಡಿ.
- ಇತರ ಜನರು ಓದಲು ಅಥವಾ ಮರು-ನಡೆಸಲು ಉದ್ದೇಶಿಸಿದ ಪ್ರಯೋಗಗಳು ಅಥವಾ ಟ್ಯೂಟೋರಿಯಲ್ಗಳನ್ನು ರಚಿಸಿ.

## ನಿರ್ಧಾರ ವೃಕ್ಷ
- ವಿನಂತಿ ಅನ್ವೇಷಣಾತ್ಮಕ, ವಿಶ್ಲೇಷಣಾತ್ಮಕ, ಅಥವಾ ಊಹಾಪೋಹ ಚಾಲಿತವಾಗಿದ್ದರೆ, `experiment` ಆಯ್ಕೆಮಾಡಿ.
- ವಿನಂತಿ ಸೂಚನಾತ್ಮಕ, ಹಂತ-ಹಂತವಾಗಿ, ಅಥವಾ ನಿರ್ದಿಷ್ಟ ಪ್ರೇಕ್ಷಣಾಂಗವನ್ನು ಗುರಿಯಾಗಿಸಿಕೊಂಡಿದ್ದರೆ, `tutorial` ಆಯ್ಕೆಮಾಡಿ.
- ಈಗಿರುವ ನೋಟ್ಬುಕ್ ಅನ್ನು ಸಂಪಾದಿಸುತ್ತಿದ್ದರೆ, ಅದನ್ನು ಪುನರ್‍ರಚನೆ ಎಂದು ಗುರುತಿಸಿ: ಉದ್ದೇಶವನ್ನು ಕಾಪಾಡಿ ಮತ್ತು ರಚನೆಯನ್ನು ಹೆಚ್ಚಿಸಿ.

## ಕೌಶಲ್ಯ ಮಾರ್ಗ (ಒಂದು ಬಾರಿ ಸೆಟ್ ಮಾಡಿ)

```bash
export CODEX_HOME="${CODEX_HOME:-$HOME/.codex}"
export JUPYTER_NOTEBOOK_CLI="$CODEX_HOME/skills/jupyter-notebook/scripts/new_notebook.py"
```

ಬಳಕೆದಾರ-ವ್ಯಾಪ್ತಿಯ ಕೌಶಲ್ಯಗಳು `$CODEX_HOME/skills` ಅಡಿಯಲ್ಲಿ ಸ್ಥಾಪನೆಯಾಗುತ್ತವೆ (ಡಿಫಾಲ್ಟ್: `~/.codex/skills`).

## ಕಾರ್ಯಪ್ರವಾಹ
1. Lock the intent.
ಉದ್ದೇಶವನ್ನು ನಿಶ್ಚಿತಪಡಿಸಿ.
ಟಿಪ್ಪಣಿಯ ವಿಧವನ್ನು ಗುರುತಿಸಿ: `experiment` ಅಥವಾ `tutorial`.
ಗುರಿ, ಪ್ರೇಕ್ಷಕರು ಮತ್ತು 'done' ಹೇಗಿರಬೇಕು ಎಂಬುದನ್ನು ದಾಖಲಿಸಿ.

2. Scaffold from the template.
ಟೆಂಪ್ಲೇಟಿನಿಂದ ಪ್ರಾಥಮಿಕ ರಚನೆ ಸೃಷ್ಟಿಸಿ.
ಹ್ಯಾಂಡ್-ಆಥರಿಂಗ್ ನಾಲ್ಕು ನೇರ JSON ಪೂರ್ವಪ್ರದೇಶದ ತಪ್ಪುಗಳನ್ನು ತಪ್ಪಿಸಲು ಸಹಾಯಕರ ಸ್ಕ್ರಿಪ್ಟ್ ಬಳಸಿ.

```bash
uv run --python 3.12 python "$JUPYTER_NOTEBOOK_CLI" \
  --kind experiment \
  --title "Compare prompt variants" \
  --out output/jupyter-notebook/compare-prompt-variants.ipynb
```

```bash
uv run --python 3.12 python "$JUPYTER_NOTEBOOK_CLI" \
  --kind tutorial \
  --title "Intro to embeddings" \
  --out output/jupyter-notebook/intro-to-embeddings.ipynb
```

3. Fill the notebook with small, runnable steps.
ಪ್ರತಿ ಕೋಡ್ ಸೆಲ್ ಅನ್ನು ಒಂದೇ ಹಂತದ ಮೇಲೆ ಫೋಕಸ್ ಆಗಿರಿಸಿ.
ಉದ್ದೇಶ ಮತ್ತು ನಿರೀಕ್ಷಿತ ಫಲಿತಾಂಶವನ್ನು ವಿವರಿಸುವ қысимир.markdown ಸರಳ ಸೆಲ್‌ಗಳನ್ನು ಸೇರಿಸಿ.
ಸಂಗ್ರಹೋತ್ಪನ್ನವಾದ ಉತ್ಪನ್ನಗಳ ತಾತ್ಕಾಲಿಕ ದೊಡ್ಡ ಶಬ್ದದ OUTPUTs ಅನ್ನು ಉದ್ಧೇಶವಾದಾಗಲಷ್ಟೇ ಬಳಸಿ; ಸ್ವಲ್ಪ ಸಾರಾಂಶ ಸಮರ್ಪಕವಾದಾಗ ದೊಡ್ಡ ಮಲಪುಗಳನ್ನು ತಪ್ಪಿಸಿ.

4. Apply the right pattern.
ಪ್ರಯೋಗಗಳಿಗೆ `references/experiment-patterns.md` ಅನುಸರಿಸಿ.
ಟ್ಯೂಟೋರಿಯಲ್ಗಾಗಿ `references/tutorial-patterns.md` ಅನುಸರಿಸಿ.

5. Edit safely when working with existing notebooks.
ಇವುಗಳ ಮೇಲೆ ಕೆಲಸ ಮಾಡುವಾಗ ನೋಟ್ಬುಕ್ ರಚನೆಯನ್ನು ಕಾಪಾಡಿ; ಮೇಲಿಂದ ಕೆಳಗೆ ಕಥಾಹೇತುವನ್ನು ಸುಧಾರಣೆ ಮಾಡುವ ಹೊರತು ಸೆಲ್‌ಗಳನ್ನು ಮರುವ್ಯವಸ್ಥೆಗೊಳಿಸುವುದನ್ನು ತಪ್ಪಿಸಿ.
ಸಂಪೂರ್ಣ ಮರುಬರಹಗಳಿಗಿಂತ ಗುರಿಮುಖ್ಯ ಸಂಪಾದನೆಗಳನ್ನು ಹೆಚ್ಚಾಗಿ ಆಯ್ಕೆಮಾಡಿ.
ನೀವು ಕಚ್ಚಾ JSON ಅನ್ನು ಸಂಪಾದಿಸಬೇಕಾದರೆ, ಮೊದಲು `references/notebook-structure.md` ಅನ್ನು ಪರಿಶೀಲಿಸಿ.

6. Validate the result.
ಪರಿಸರವು ಅನುಮತಿಸಿದಾಗ, ನೋಟ್ಬುಕ್ ಅನ್ನು ಮೇಲಿನಿಂದ ಕೆಳಗೆ ಚಾಲನೆ ಮಾಡಿ.
ನಡೆಸಲು ಸಾಧ್ಯವಿಲ್ಲದಿದ್ದರೆ, ಅದನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ಉಲ್ಲೇಖಿಸಿ ಮತ್ತು ಸ್ಥಳೀಯವಾಗಿ ಮಾನ್ಯಗೊಳಿಸಲು ಹೇಗೆ ಎಂದು ಸೂಚಿಸಿ.
ಫೈನಲ್ ಪಾಸ್ ಚೆಕ್‌ಲಿಸ್ಟ್ `references/quality-checklist.md` ನಲ್ಲಿ ಬಳಸಿ.

## ಟೆಂಪ್ಲೇಟ್ಗಳು ಮತ್ತು ಸಹಾಯಕರ ಸ್ಕ್ರಿಪ್ಟ್
- ಟೆಂಪ್ಲೇಟ್ಗಳು `assets/experiment-template.ipynb` ಮತ್ತು `assets/tutorial-template.ipynb` ನಲ್ಲಿ ಇವೆ.
- ಸಹಾಯಕರ ಸ್ಕ್ರಿಪ್ಟ್ ಒಂದು ಟೆಂಪ್ಲೇಟ್ನ್ನು ಲೋಡ್ ಮಾಡುತ್ತದೆ, ಶೀರ್ಷಿಕೆ ಸೆಲ್ ಅನ್ನು 업데이트 ಮಾಡುತ್ತದೆ, ಮತ್ತು ಒಂದು ನೋಟ್ಬುಕ್ ಬರೆಯುತ್ತದೆ.

ಸ್ಕ್ರಿಪ್ಟ್ ದಾರಿ:
- `$JUPYTER_NOTEBOOK_CLI` (ಸ್ಥಾಪಿತ ಡಿಫಾಲ್ಟ್: `$CODEX_HOME/skills/jupyter-notebook/scripts/new_notebook.py`)

## ತಾತ್ಕಾಲಿಕ ಮತ್ತು ಔಟ್‌ಪುಟ್ ಸಂಪ್ರದಾಯಗಳು
- ಮಧ್ಯವರ್ತಿ ಫೈಲ್‌ಗಳಿಗಾಗಿ `tmp/jupyter-notebook/` ಬಳಸಿರಿ; ಕೆಲಸ ಮುಗಿಸಿದ ನಂತರ ಅಳಿಸಿ.
- ಈ ರೆಪೊನಲ್ಲಿ ಕೆಲಸ ಮಾಡೋರು ಅಂತಿಮ ಕಲಾಪಗಳನ್ನು `output/jupyter-notebook/` ಅಡಿಯಲ್ಲಿ ಬರೆಯಿರಿ.
- ಸ್ಥಿರ, ವಿವರಣಾತ್ಮಕ ಫೈಲ್-ಹೆಸರುಗಳನ್ನು ಬಳಸಿ (ಉದಾಹರಣೆಗೆ, `ablation-temperature.ipynb`).

## Dependencies (install only when needed)
ಆಧಾರ ನಿರ್ವಹಣೆಗೆ `uv` ಅನ್ನು ಪ್ರಾಧಾನ್ಯವಾಗಿ ಬಳಸಿ.

ಸ್ಥಳೀಯ ನೋಟ್ಬುಕ್ ನಿರ್ವಹಣೆಗೆ ಐಚ್ಛಿಕ Python ಪ್ಯಾಕೇಜ್‌ಗಳು:

```bash
uv pip install jupyterlab ipykernel
```

ಬಂಡಲ್ ಮಾಡಲಾದ scaffold ಸ್ಕ್ರಿಪ್ಟ್ ಕೇವಲ Python ಮಾನದಂಡ ಪುಸ್ತಕಾಲಯವನ್ನು ಬಳಸುತ್ತದೆ ಮತ್ತು ಹೆಚ್ಚುವರಿ ಅವಲಂಬನೆಗಳನ್ನು ಅಗತ್ಯವಿಲ್ಲ.

## ಪರಿಸರ
ಯಾವುದೇ ಅಗತ್ಯ ವಾತಾವರಣ ಚರಗಳಿಲ್ಲ.

## ರೆಫರೆನ್ಸ್ ನಕ್ಷೆ
- `references/experiment-patterns.md`: ಪ್ರಯೋಗದ ರಚನೆ ಮತ್ತು ಸೂತ್ರಶಾಸ್ತ್ರಗಳು.
- `references/tutorial-patterns.md`: ಟ್ಯೂಟೋರಿಯಲ್ ರಚನೆ ಮತ್ತು ಬೋಧನಾ ಹರಿವು.
- `references/notebook-structure.md`: ನೋಟ್ಬುಕ್ JSON ರೂಪ ಮತ್ತು ಸುರಕ್ಷಿತ ಸಂಪಾದನೆ ನಿಯಮಗಳು.
- `references/quality-checklist.md`: ಅಂತಿಮ ಮಾನ್ಯತಾ ಚೆಕ್‌ಲಿಸ್ಟ್.
