# नमुना रसिद फिक्स्चरहरू

नोटकबुक चलाउनु बिना निरीक्षणको लागि तीन पूर्व-उत्पादित रसिद फाइलहरू।

| फाइल | के हो |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` उपकरण कलको लागि मान्य हस्ताक्षर गरिएको रसिद। प्रमाणीकरणले True फर्काउँछ। |
| `02_tampered_receipt.json` | त्यहि रसिद, हस्ताक्षर पछि एउटा फिल्ड परिवर्तन गरिएको। प्रमाणीकरणले False फर्काउँछ। |
| `03_chain_three_receipts.json` | तीन मान्य रसिदहरूको श्रृंखला (search, hold, book) जसमा `previous_receipt_hash` हरेकलाई अघिल्लोसँग जोड्दछ। |

## नमूनाहरूको प्रमाणीकरण

नोटकबुकले चार खण्डहरूमा प्रमाणीकरण प्रक्रिया देखाउँछ। यी फिक्स्चरहरूलाई 
नोटकबुक कथा बिना सिधै प्रमाणीकरण गर्न:

```python
import json
from pathlib import Path

# तपाईंले आयात र सहयोगी फंक्शनहरू पूरा गरेको मानिन्छ
# १८-साइन्ड-रसिदहरू.ipynb को भाग १ र २ बाट।

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # सत्य

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # असत्य

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

स्थिर टाइमस्ट्याम्पहरू प्रयोग गरेर बाइट-फेरि पुन:निर्माणयोग्यता को लागि। पुनः उत्पन्न गर्न:


## विद्यार्थीहरूले कच्चा JSON निरीक्षणबाट के सिक्छन्

कच्चा रसिद ढाँचा पढ्दा त्यस्तो सहज ज्ञान बढ्छ जुन नोटकबुकका सेलहरूले सधैं 
दिइदैनन्। JSON छिटो हेर्ने विद्यार्थीहरूले प्रायः देख्दछन्:

1. हस्ताक्षर एक अपारदर्शी base64url स्ट्रिङ हो, तर अन्य सबै फिल्डहरू सामान्य पढ्न सकिने JSON हुन्छ। हस्ताक्षरले सामग्रीलाई सङ्कुचित गर्दैन; यसले त्यसलाई प्रमाणित गर्छ।
2. `public_key` रसिदमा समावेश गरिएको हुन्छ। प्रमाणीकारकलाई अन्य केही चाहिँदैन (यो कुञ्जी साँच्चिकै दाबी गरिएको जारीकर्ता सँग सम्बन्धित छ भन्ने विश्वास गर्नु पर्ने विषय; पहिचान पूर्वाधारमा पढाइ README हेर्नुहोस्)।
3. कुनै पनि फिल्डको एउटा अक्षर परिमार्जन गरी, त्यसपछि यो फाइललाई `02_tampered_receipt.json` सँग पुनः तुलना गर्दा, बाइट-स्तरको क्रिया प्रक्रियालाई स्पष्ट बनाउन मद्दत हुन्छ।
