# Агенты ИИ для начинающих — учебное пособие и конспект курса

This guide provides a summary of the "AI Agents for Beginners" course and explains key concepts, frameworks, and design patterns for building AI Agents.

## 1. Введение в агентов ИИ

**Что такое агенты ИИ?**
Агенты ИИ — это системы, которые расширяют возможности больших языковых моделей (LLMs), предоставляя им доступ к **инструментам**, **знаниям** и **памяти**. В отличие от стандартного чат-бота на базе LLM, который просто генерирует текст на основе обучающих данных, агент ИИ может:
- **Воспринимать** своё окружение (через сенсоры или входные данные).
- **Размышлять** о том, как решить задачу.
- **Действовать**, чтобы изменить окружение (через актуаторы или выполнение инструментов).

**Ключевые компоненты агента:**
- **Окружение**: Пространство, в котором действует агент (например, система бронирования).
- **Сенсоры**: Механизмы для сбора информации (например, чтение API).
- **Актуаторы**: Механизмы для выполнения действий (например, отправка письма).
- **Мозг (LLM)**: Двигатель рассуждения, который планирует и решает, какие действия предпринимать.

## 2. Agentic Frameworks

The course uses **Microsoft Agent Framework (MAF)** with **Azure AI Foundry Agent Service V2** for building agents:

| Компонент | Фокус | Лучше всего подходит для |

## 3. Agentic Design Patterns

Паттерны проектирования помогают структурировать работу агентов для надёжного решения задач.

### **Tool Use Pattern** (Lesson 4)
Этот паттерн позволяет агентам взаимодействовать с внешним миром.
- **Концепция**: агенту предоставляется "схема" (список доступных функций и их параметров). LLM решает, *какой* инструмент вызвать и с *какими* аргументами на основе запроса пользователя.
- **Поток**: Запрос пользователя -> LLM -> **Выбор инструмента** -> **Выполнение инструмента** -> LLM (с результатом инструмента) -> Окончательный ответ.
- **Сценарии использования**: получение данных в реальном времени (погода, цены на акции), выполнение вычислений, запуск кода.

### **Planning Pattern** (Lesson 7)
Этот паттерн позволяет агентам решать сложные многозадачные задачи.
- **Концепция**: агент разбивает высокоуровневую цель на последовательность более мелких подзадач.
- **Подходы**:
  - **Разбиение задач (Task Decomposition)**: разделение «Спланировать поездку» на «Забронировать рейс», «Забронировать отель», «Арендовать автомобиль».
  - **Итеративное планирование (Iterative Planning)**: переоценка плана на основе результатов предыдущих шагов (например, если рейс заполнен, выбрать другую дату).
- **Реализация**: часто включает агента‑«Планировщика», который генерирует структурированный план (например, JSON), который затем выполняют другие агенты.

## 4. Принципы проектирования

При проектировании агентов учитывайте три измерения:
- **Пространство**: агенты должны связывать людей и знания, быть доступными, но ненавязчивыми.
- **Время**: агенты должны учиться на *прошлом*, давать уместные подсказки *сейчас* и адаптироваться для *будущего*.
- **Ядро**: принимать неопределённость, но устанавливать доверие через прозрачность и контроль пользователя.

## 5. Краткое изложение ключевых уроков

- **Урок 1**: Агенты — это системы, а не просто модели. Они воспринимают, рассуждают и действуют.
- **Урок 2**: Microsoft Agent Framework абстрагирует сложность вызова инструментов и управления состоянием.
- **Урок 3**: Проектируйте с учётом прозрачности и контроля со стороны пользователя.
- **Урок 4**: Инструменты — это «руки» агента. Определение схемы имеет решающее значение для того, чтобы LLM поняла, как ими пользоваться.
- **Урок 7**: Планирование — это «исполнительная функция» агента, позволяющая ему справляться со сложными рабочими процессами.
