# Sample Receipt Fixtures

ไฟล์ใบเสร็จที่สร้างไว้ล่วงหน้าสามไฟล์สำหรับตรวจสอบโดยไม่ต้องรันโน้ตบุ๊ก

| File | สิ่งที่เป็น |
|---|---|
| `01_valid_receipt.json` | ใบเสร็จที่ลงลายมือชื่อถูกต้องสำหรับการเรียกใช้เครื่องมือ `lookup_flights` การตรวจสอบจะคืนค่า True |
| `02_tampered_receipt.json` | ใบเสร็จเดียวกันแต่มีการแก้ไขหนึ่งช่องข้อมูลหลังจากลงลายมือชื่อ การตรวจสอบจะคืนค่า False |
| `03_chain_three_receipts.json` | โซ่ของใบเสร็จที่ถูกต้องสามใบ (ค้นหา, ถือครอง, จอง) โดยใช้ `previous_receipt_hash` เชื่อมโยงแต่ละใบกับใบก่อนหน้า |

## การตรวจสอบตัวอย่าง

โน้ตบุ๊กจะสาธิตการตรวจสอบในสี่ส่วน เพื่อยืนยันข้อมูลตัวอย่างเหล่านี้โดยตรงโดยไม่ต้องรันตามเนื้อหาในโน้ตบุ๊ก:

```python
import json
from pathlib import Path

# สมมติว่าคุณได้ทำการนำเข้าและฟังก์ชันช่วยเหลือเสร็จสิ้นแล้ว
# จากส่วนที่ 1 และ 2 ของ 18-signed-receipts.ipynb

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # จริง

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # เท็จ

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## วิธีการสร้างไฟล์เหล่านี้

ไฟล์เหล่านี้ใช้เส้นทางโค้ดเดียวกับโน้ตบุ๊ก โดยใช้กุญแจลงลายมือชื่อคงที่
และเวลาที่คงที่เพื่อให้ผลได้เหมือนเดิมในระดับไบต์ หากต้องการสร้างใหม่:

```bash
python3 generate_fixtures.py
```

(สคริปต์อยู่ที่ `generate_fixtures.py` ในไดเรกทอรีนี้)

## สิ่งที่นักเรียนได้เรียนรู้จากการตรวจสอบ JSON ดิบ

การอ่านรูปแบบใบเสร็จดิบช่วยสร้างความเข้าใจที่เซลล์ในโน้ตบุ๊กบางครั้งให้ไม่ได้ นักเรียนที่แค่สแกน JSON มักสังเกตเห็นว่า:

1. ลายมือชื่อเป็นสตริง base64url แบบไม่เปิดเผยข้อมูล แต่ช่องข้อมูลอื่นๆ เป็น JSON ที่อ่านได้อย่างชัดเจน ลายมือชื่อไม่ได้เข้ารหัสเนื้อหา แต่เป็นการรับรองความถูกต้อง
2. `public_key` ถูกฝังอยู่ในใบเสร็จ ผู้ตรวจสอบไม่จำเป็นต้องใช้อะไรเพิ่มเติมเพื่อยืนยัน (ขึ้นอยู่กับการไว้วางใจว่ากุญแจนั้นเป็นของผู้ออกใบเสร็จจริง; ดูใน README บทเรียนเรื่องโครงสร้างพื้นฐานด้านตัวตน)
3. การแก้ไขเพียงตัวอักษรเดียวในช่องข้อมูลใดๆ แล้วนำไปเปรียบเทียบกับไฟล์ `02_tampered_receipt.json` ทำให้กลไกระดับไบต์นั้นชัดเจนขึ้นมาก
