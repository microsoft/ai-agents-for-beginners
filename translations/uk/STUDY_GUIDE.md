# Агенти ШІ для початківців - Навчальний посібник і конспект курсу

Цей посібник надає резюме курсу "AI Agents for Beginners" і пояснює ключові концепції, фреймворки та патерни проєктування для створення агентів ШІ.

## 1. Вступ до агентів ШІ

**Що таке агенти ШІ?**
Агенти ШІ — це системи, які розширюють можливості великих мовних моделей (LLM), надаючи їм доступ до **інструментів**, **знань** та **пам'яті**. На відміну від стандартного чатбота на базі LLM, який лише генерує текст на основі навчальних даних, агент ШІ може:
- **Сприймати** своє середовище (через датчики або вхідні дані).
- **Міркувати** про те, як вирішити задачу.
- **Діяти**, щоб змінити середовище (через виконавчі механізми або виконання інструментів).

**Ключові компоненти агента:**
- **Навколишнє середовище**: Простір, де працює агент (наприклад, система бронювання).
- **Датчики**: Механізми для збору інформації (наприклад, читання API).
- **Виконавчі механізми**: Механізми для виконання дій (наприклад, відправка електронної пошти).
- **Мозок (LLM)**: Двигун міркувань, що планує та вирішує, які дії виконувати.

## 2. Фреймворки для агентів

Курс використовує **Microsoft Agent Framework (MAF)** з **Azure AI Foundry Agent Service V2** для створення агентів:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | Уніфікований SDK для Python/C# для агентів, інструментів та робочих процесів | Створення агентів з інструментами, багатоагентних робочих процесів та виробничих шаблонів. |
| **Azure AI Foundry Agent Service** | Кероване хмарне середовище виконання | Безпечне, масштабоване розгортання з вбудованим управлінням станом, спостережливістю та довірою. |

## 3. Патерни проєктування агентів

Патерни проєктування допомагають структурувати роботу агентів для надійного вирішення задач.

### **Патерн використання інструментів** (Урок 4)
Цей патерн дозволяє агентам взаємодіяти з зовнішнім світом.
- **Концепція**: Агенту надається "схему" (список доступних функцій та їхніх параметрів). LLM вирішує *який* інструмент викликати і з *якими* аргументами на основі запиту користувача.
- **Flow**: Запит користувача -> LLM -> **Вибір інструменту** -> **Виконання інструменту** -> LLM (з результатом інструменту) -> Кінцева відповідь.
- **Випадки використання**: Отримання даних у реальному часі (погода, ціни акцій), виконання обчислень, запуск коду.

### **Патерн планування** (Урок 7)
Цей патерн дозволяє агентам розв'язувати складні багатокрокові завдання.
- **Концепція**: Агент розбиває високорівневу мету на послідовність менших підзадач.
- **Підходи**:
  - **Декомпозиція задач**: Розбиття "Спланувати поїздку" на "Забронювати рейс", "Забронювати готель", "Орендувати авто".
  - **Ітеративне планування**: Переоцінювання плану на основі результатів попередніх кроків (наприклад, якщо рейс заповнений, обрати іншу дату).
- **Реалізація**: Часто включає агента «Планувальник», який генерує структурований план (наприклад, JSON), що потім виконується іншими агентами.

## 4. Принципи проєктування

Під час проєктування агентів враховуйте три виміри:
- **Space**: Агенти повинні поєднувати людей і знання, бути доступними, але ненав'язливими.
- **Time**: Агенти повинні вчитися на *Минулому*, надавати доречні підказки в *Теперішньому* і адаптуватися для *Майбутнього*.
- **Core**: Приймайте невизначеність, але встановлюйте довіру через прозорість і контроль користувача.

## 5. Підсумок ключових уроків

- **Lesson 1**: Агенти — це системи, а не лише моделі. Вони сприймають, міркують і діють.
- **Lesson 2**: Microsoft Agent Framework абстрагує складність виклику інструментів та управління станом.
- **Lesson 3**: Проєктуйте з урахуванням прозорості та контролю користувача.
- **Lesson 4**: Інструменти — це «руки» агента. Визначення схеми є вирішальним для того, щоб LLM розумів, як їх використовувати.
- **Lesson 7**: Планування — це «виконавча функція» агента, що дозволяє йому справлятися зі складними робочими процесами.
