# AI ایجنٹس برائے ابتدائی افراد - مطالعہ رہنما اور کورس کا خلاصہ

یہ رہنما "AI Agents for Beginners" کورس کا خلاصہ فراہم کرتی ہے اور AI ایجنٹس بنانے کے لیے کلیدی تصورات، فریم ورکس، اور ڈیزائن پیٹرنز کی وضاحت کرتی ہے۔

## 1. AI ایجنٹس کا تعارف

**AI ایجنٹس کیا ہیں؟**
AI ایجنٹس ایسے سسٹمز ہیں جو بڑے لسانی ماڈلز (LLMs) کی صلاحیتوں کو **اوزار**، **علم**، اور **میموری** تک رسائی دے کر بڑھاتے ہیں۔ ایک معیاری LLM چیٹ بوٹ جو صرف تربیتی ڈیٹا کی بنیاد پر متن پیدا کرتا ہے، کے برعکس، ایک AI ایجنٹ کر سکتا ہے:
- **محسوس** کرے اپنا ماحول (سینسرز یا ان پٹس کے ذریعے)۔
- **عقل مندی** سے مسئلے کو حل کرنے کے بارے میں سوچے/منطقی نتیجہ اخذ کرے۔
- **عمل** کرے تاکہ ماحول کو تبدیل کرے (ایکچیویٹرز یا اوزاروں کے نفاذ کے ذریعے)۔

**ایجنٹ کے کلیدی اجزاء:**
- **ماحول**: وہ جگہ جہاں ایجنٹ کام کرتا ہے (مثلاً ایک بکنگ سسٹم)۔
- **سینسرز**: معلومات اکٹھا کرنے کے طریقے (مثلاً ایک API پڑھنا)۔
- **ایکچیویٹرز**: اقدامات انجام دینے کے طریقے (مثلاً ای میل بھیجنا)۔
- **دماغ (LLM)**: سوچنے والا انجن جو منصوبہ بناتا ہے اور فیصلہ کرتا ہے کہ کون سے اقدامات لینے ہیں۔

## 2. Agentic فریم ورکس

کورس ایجنٹس بنانے کے لیے **Microsoft Agent Framework (MAF)** اور **Azure AI Foundry Agent Service V2** استعمال کرتا ہے:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | متحدہ Python/C# SDK برائے ایجنٹس، اوزار، اور ورک فلو | اوزاروں کے ساتھ ایجنٹس بنانا، کثیر ایجنٹ ورک فلو، اور پروڈکشن پیٹرنز کے لیے۔ |
| **Azure AI Foundry Agent Service** | مینیجڈ کلاؤڈ رن ٹائم | محفوظ، وسعت پذیر تعیناتی جس میں بلٹ ان اسٹیٹ مینجمنٹ، مشاہداتی صلاحیت، اور اعتماد شامل ہیں۔ |

## 3. Agentic ڈیزائن پیٹرنز

ڈیزائن پیٹرنز اس بات میں مدد دیتے ہیں کہ ایجنٹس مسئلے کو قابل اعتماد انداز میں حل کرنے کے لیے کیسے کام کریں۔

### **Tool Use Pattern** (Lesson 4)
یہ پیٹرن ایجنٹس کو بیرونی دنیا کے ساتھ بات چیت کرنے کے قابل بناتا ہے۔
- **تصور**: ایجنٹ کو ایک "schema" (دستیاب فنکشنز اور ان کے پیرامیٹرز کی فہرست) فراہم کی جاتی ہے۔ LLM فیصلہ کرتا ہے کہ صارف کی درخواست کی بنیاد پر *کون سا* ٹول کال کرنا ہے اور *کس* دلیل کے ساتھ۔
- **فلو**: صارف کی درخواست -> LLM -> **اوزار کا انتخاب** -> **اوزار کی انجام دہی** -> LLM (اوزار کے نتیجے کے ساتھ) -> حتمی جواب۔
- **استعمال کے معاملات**: حقیقی وقت کا ڈیٹا حاصل کرنا (موسم، اسٹاک قیمتیں)، حسابات انجام دینا، کوڈ چلانا۔

### **Planning Pattern** (Lesson 7)
یہ پیٹرن ایجنٹس کو پیچیدہ، متعدد مراحل والے کام حل کرنے کے قابل بناتا ہے۔
- **تصور**: ایجنٹ ایک اعلیٰ سطحی ہدف کو چھوٹے ذیلی کاموں کی ترتیب میں تقسیم کرتا ہے۔
- **رویہ**:
  - **ٹاسک ڈی کامپوزیشن**: "سفر کی منصوبہ بندی" کو "پرواز بک کریں"، "ہوٹل بک کریں"، "کار کرایہ پر لیں" میں تقسیم کرنا۔
  - **تکراری منصوبہ بندی**: پچھلے اقدامات کے نتائج کی بنیاد پر منصوبے کا دوبارہ جائزہ لینا (مثلاً اگر پرواز بھر چکی ہو تو مختلف تاریخ منتخب کرنا)۔
- **عملدرآمد**: اکثر ایک "پلانر" ایجنٹ شامل ہوتا ہے جو ایک منظم منصوبہ (مثلاً JSON) تیار کرتا ہے اور پھر دوسرے ایجنٹس اس منصوبے کو نافذ کرتے ہیں۔

## 4. ڈیزائن کے اصول

جب ایجنٹس ڈیزائن کریں تو تین بُعدوں پر غور کریں:
- **جگہ (Space)**: ایجنٹس کو لوگوں اور علم کو جوڑنا چاہیے، قابل رسائی مگر غیر مداخلت رکھنا چاہیے۔
- **وقت (Time)**: ایجنٹس کو *ماضی* سے سیکھنا چاہیے، *حال* میں مناسب اشارے دینے چاہئیں، اور *مستقبل* کے لیے خود کو ڈھالنا چاہیے۔
- **جوہر (Core)**: غیر یقینی صورتحال کو قبول کریں مگر شفافیت اور صارف کنٹرول کے ذریعے اعتماد قائم کریں۔

## 5. اہم اسباق کا خلاصہ

- **Lesson 1**: ایجنٹس سسٹمز ہیں، صرف ماڈلز نہیں۔ وہ محسوس کرتے ہیں، سوچتے ہیں، اور عمل کرتے ہیں۔
- **Lesson 2**: Microsoft Agent Framework اوزار کال کرنے اور اسٹیٹ مینجمنٹ کی پیچیدگی کو خلاصہ کرتا ہے۔
- **Lesson 3**: شفافیت اور صارف کنٹرول کو مدنظر رکھتے ہوئے ڈیزائن کریں۔
- **Lesson 4**: اوزار ایجنٹ کے "ہاتھ" ہیں۔ اسکیمہ کی تعریف LLM کے لیے یہ سمجھنے میں ضروری ہے کہ انہیں کیسے استعمال کرنا ہے۔
- **Lesson 7**: منصوبہ بندی ایجنٹ کا "انتظامی فنکشن" ہے، جو اسے پیچیدہ ورک فلو سے نمٹنے کے قابل بناتا ہے۔
