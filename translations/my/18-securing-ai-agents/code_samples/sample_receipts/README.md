# နမူနာ အကြောင်းကြားလွှာ ဖိုင်များ

မှတ်စုစာတမ်းမဖွင့်ပဲ စစ်ဆေးနိုင်ရန် အနောက်ဆုံး သတ်မှတ်ထားသော အကြောင်းကြားလွှာ ဖိုင်သုံးဖိုင်။

| ဖိုင် | အရာသည် ဘာလဲ |
|---|---|
| `01_valid_receipt.json` | `lookup_flights` ကိရိယာခေါ်ဆိုမှုအတွက် မှန်ကန်စစ်မှတ်ဆိုင်သော အကြောင်းကြားလွှာ တစ်စောင်။ အတည်ပြုခြင်းသည် True ဖြစ်သည်။ |
| `02_tampered_receipt.json` | စာချုပ်ဆိုင်ရာ အကြောင်းကြားလွှာနှင့် တူသော အကြောင်းကြားလွှာတစ်စောင်၊ သက်ဆိုင်ရာ အကွက်တစ်ခုကို လက်မှတပ်ပြီး ပြင်ဆင်ထားသည်။ အတည်ပြုခြင်းသည် False ဖြစ်သည်။ |
| `03_chain_three_receipts.json` | မှန်ကန်သော အကြောင်းကြားလွှာသုံးစောင် (ရှာဖွေမှု၊ မှတ်ပုံတင်မှု၊ လက်မှတ်ရေးထိုးမှု) ကို `previous_receipt_hash` ဖြင့် နှစ်သိမ့်အဆက်အဆက် ချိတ်ဆက်ထားသည်။ |

## နမူနာများကို အတည်ပြုခြင်း

မှတ်စုစာတမ်းသည် အတည်ပြုခြင်းကို ငါးပုဒ်ကို ဖြတ်သန်း သွားပါသည်။ ဒီဖိုင်တွေကိုမှတ်စု စာတမ်းအပြောအဆိုမခံဘဲ တိုက်ရိုက် အတည်ပြုရန်:

```python
import json
from pathlib import Path

# သင်သည် 18-signed-receipts.ipynb ၏ အခန်း ၁ နှင့် ၂ တွင် ရှိသော imports နှင့် အကူအညီလုပ်ဆောင်ချက်များကို ပြီးမြောက်စွာ ဆောင်ရွက်ထားသည်ဟု ထင်မှတ်သည်။
# 18-signed-receipts.ipynb ၏ အခန်း ၁ နှင့် ၂ မှ။

valid = json.loads(Path("01_valid_receipt.json").read_text())
print(f"Valid receipt: {verify_receipt(valid)}")        # မှန်ကန်သည်။

tampered = json.loads(Path("02_tampered_receipt.json").read_text())
print(f"Tampered receipt: {verify_receipt(tampered)}")  # မှားသည်။

chain = json.loads(Path("03_chain_three_receipts.json").read_text())
for r in verify_chain(chain):
    print(f"  Receipt {r['index']} ({r['tool']}): {'VALID' if r['overall_valid'] else 'INVALID'}")
```

## ဒီတွေ ဘယ်လို ပြုလုပ်ခဲ့သလဲ

ဒီဖိုင်များသည် မှတ်စုစာတမ်းတွေနဲ့ တူညီသော ကုဒ်လမ်းကြောင်းကို အသုံးပြုပြီး၊ တစ်စိတ်တစ်ပိုင်းသော လက်မှတ် ဆိုင်းငံ့သော သော့ နှင့် စက္ကန့်အချိန်များကို အတည်ပြုထားသောကြောင့် bytes မပြောင်းလဲသော အခြေအနေဖြစ်သည်။ ပြန်လည်ဖန်တီးရန်:

```bash
python3 generate_fixtures.py
```

(ဒီ စာတိုက်တွင်းရှိ `generate_fixtures.py` မှာ စာရေးထားသည်။)

## ကျောင်းသားများ အတွက် အမှန် JSON ကို ကြည့်ရှုခြင်းမှ သင်ယူသော အရာများ

အမှန်ကြောင်းက အကြောင်းကြားလွှာ ဖော်ပြချက်ကို ဖတ်ခြင်းဖြင့် မှတ်စုစာတမ်းထဲက cells များထက် အတွေးအမြင် ပိုမို ရရှိစေပါသည်။ JSON ကို မကြည့်လျှင် မမြင်ရနိုင်သလို၊

1. လက်မှတ်သည် မမြင်သာတဲ့ base64url string ဖြစ်ပေမယ့် အခြား အကွက်အားလုံးသည် အမြင်သာသော JSON ဖြစ်သည်။ လက်မှတ်သည် content ကို ကွပ်ကန်းခြင်းမပြုသော်လည်း ပြသထောက်ခံသည်။
2. `public_key` ကို အကြောင်းကြားလွှာထဲမှာ ထည့်ထားသည်။ စစ်ဆေးသူတစ်ဦးအနေနဲ့ ထုတ်ပြန်သူက ဆင့်ကဲမှုရှိသည်ဟုယုံကြည်သည့်အခါ၌ လက်ရှိသော စားပွဲတွင် အသေးစိတ် ထည့်သွင်းစစ်ဆေးရန် မလိုပါ။
3. အကွက်တစ်ခုလုံးတွင် စာလုံးတစ်လုံးကို ပြောင်းလဲပြီး `02_tampered_receipt.json` နှင့် နှိုင်းယှဉ်သည့်အခါ bytes အဆင့်နှင့်စနစ်ကို ပိုမိုသေချာလာသည်။
