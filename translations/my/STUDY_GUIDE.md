# AI Agents for Beginners - ပညာသင်လမ်းညွှန်နှင့် သင်ခန်းစာ အကျဉ်းချုပ်

ဤလမ်းညွှန်သည် "AI Agents for Beginners" သင်ခန်းစာ၏ အကျဉ်းချုပ်ကို ပေးပြီး AI Agent များ တည်ဆောက်ရာတွင် အဓိက ဖြစ်သော အကြောင်းအရာများ၊ ဖရိမ်ဝေါ့များနှင့် ဒီဇိုင်းပုံစံများကို ရှင်းပြသည်။

## 1. AI Agents မိတ်ဆက်ခြင်း

**AI Agents ဆိုတာဘာလဲ?**  
AI Agents သည် Large Language Models (LLMs) ၏ စွမ်းအားများကို **ကိရိယာများ၊ သိပ္ပံပညာနှင့် မှတ်ဉာဏ်** များ အသုံးပြုခြင်းဖြင့် တိုးချဲ့ပေးသည့် စနစ်များဖြစ်သည်။ training data ကို အခြေခံ၍ စာသား ကိုယ်တိုင်ထုတ်ပေးသော စံပုံ LLM chatbot တစ်ခု အနေဖြင့် မဟုတ်ဘဲ AI Agent သည် -  
- ပတ်ဝန်းကျင်ကို **ခံစား** နိုင်သည် (ဆင်ဆာ သို့မဟုတ် input များမှတဆင့်)။  
- ပြဿနာဖြေရှင်းရန် **အကြံဉာဏ်နားလည်** ပြုသည်။  
- ပတ်ဝန်းကျင်ကိုပြောင်းလဲရန် **လုပ်ဆောင်** များဆောင်ရွက်သည် (actuators သို့မဟုတ် ကိရိယာများ အသုံးပြု၍)။

**Agent ၏ အဓိကအစိတ်အပိုင်းများ**  
- **ပတ်ဝန်းကျင်**: Agent လည်ပတ်ရာ နေရာ (ဥပမာ - ဘွတ်ကင်စနစ်)  
- **ဆင်ဆာများ**: အချက်အလက်စုဆောင်းရာ ကိရိယာများ (ဥပမာ - API ဖတ်ခြင်း)  
- **Actuators**: လုပ်ဆောင်ချက်များကို တင်ပြရန် ကိရိယာများ (ဥပမာ - အီးမေးလ်ပို့ခြင်း)  
- **ဦးနှောက် (LLM)**: လုပ်ဆောင်ရန် အစီအစဉ်ချပြီး ဆုံးဖြတ်ချက်ယူသော အင်ဂျင်

## 2. Agentic Frameworks

သင်ခန်းစာတွင် Agent များတည်ဆောက်ရန်အတွက် **Microsoft Agent Framework (MAF)** နှင့် **Azure AI Foundry Agent Service V2** ကို အသုံးပြုသည် -

| အစိတ်အပိုင်း | အာရုံစိုက်မှု | အတွက်အကောင်းဆုံး |  
|-----------|-------|----------|  
| **Microsoft Agent Framework** | Agent များ၊ ကိရိယာများနှင့် workflow များအတွက် စုပေါင်း Python/C# SDK | ကိရိယာများဖြင့် Agent များတည်ဆောက်ခြင်း၊ multi-agent workflow များနှင့် ထုတ်လုပ်မှုပုံစံများ |  
| **Azure AI Foundry Agent Service** | စီမံခန့်ခွဲထားသည့် cloud runtime | အပြင်းအထန်လုံခြုံ၍ state management၊ observability နှင့် ယုံကြည်မှုတို့ပါရှိသည့် ကြီးကြပ်မှု |

## 3. Agentic Design Patterns

ဒီဇိုင်းပုံစံများသည် agent များ အကြောင်းပြု၍ ပြဿနာကို ယုံကြည်စိတ်ချစွာ ဖြေရှင်းနိုင်ရန် အလုပ်လုပ်ပုံကို ဖွဲ့စည်းပေးသည်။

### **Tool Use Pattern** (သင်ခန်းစာ 4)  
ဤ pattern သည် agent များကို ပြင်ပကမ္ဘာနှင့် ဆက်သွယ်နိုင်ရန် ခွင့်ပြုသည် -  
- **အယူအဆ**: Agent သည် "schema" (အသုံးပြုနိုင်သော function များနှင့် parameter များစာရင်း) တစ်ခုကို လက်ခံရရှိသည်။ LLM သည် အသုံးပြုသူ တောင်းဆိုချက် အရ *ဘယ် tool* ကို သုံးမည်နှင့် *ဘာ parameter* ဖြင့် ခေါ်မည်ကို ဆုံးဖြတ်သည်။  
- **လည်ပတ်မှုစဉ်** - အသုံးပြုသူ တောင်းဆိုချက် -> LLM -> **Tool ရွေးချယ်မှု** -> **Tool အကောင်အထည်ဖော်မှု** -> LLM (tool output နှင့်အတူ) -> နောက်ဆုံး အဖြေ။  
- **အသုံးပြုမှု**: အချိန်နှင့်တပြေးညီ ဒေတာ ရှာဖွေခြင်း (ရာသီဥတု၊ စတော့)၊ တွက်ချက်မှုများ ဆောင်ရွက်ခြင်း၊ ကုဒ် လုပ်ဆောင်ခြင်း။

### **Planning Pattern** (သင်ခန်းစာ 7)  
ဤ pattern သည် agent များကို ခက်ခဲသည့်၊ အဆင့်များစုံ task များကို ဖြေရှင်းနိုင်ရန် ခွင့်ပြုသည် -  
- **အယူအဆ**: Agent သည် အကြီးစားရည်မှန်းချက်ကို အသေးစား subtasks များအဖြစ် ကွဲထွက်စီမံသည်။  
- **နည်းလမ်းများ**:  
  - **Task Decomposition**: "ခရီးစဉ်စီမံခြင်း" ကို "လေယာဉ်ဘတ်"၊ "ဟိုတယ်ဘတ်"၊ "ကားငှား" နေရာများသို့ ခွဲခြမ်းခြင်း  
  - **Iterative Planning**: အရင်အဆင့်များ ရလဒ်အပေါ်မူတည်၍ ပြန်လည်သုံးသပ်ခြင်း (ဥပမာ - လေယာဉ်ပြည့်နေပါက အခြားရက်ရွေးခြင်း)  
- **အကောင်အထည်ဖော်မှု**: "Planner" agent တစ်ခုဖြင့် တဖန်ဖန် စီမံထားသော ရှင်းလင်းမြင်သာ JSON ပုံစံ အစီအစဉ် တစ်ခု ရေးဆွဲပြီး အခြား Agent များအကောင်အထည်ဖော်သည့် ပုံစံ ဖြစ်သည်။

## 4. ဒီဇိုင်းအခြေခံကိစ္စများ

Agent များ ဒီဇိုင်းရေးဆွဲရာတွင် ဖြတ်သန်းသင့်သည့် အတိုင်းအတာ ၃ ချက် -  
- **နေရာ**: Agent များသည် လူများနှင့် သိပ္ပံပညာများကို ချိတ်ဆက်ပေးသင့်ပြီး လွယ်ကူရောက်ရှိနိုင်ပေမယ့် မတက်ကြွစေရန်။  
- **အချိန်**: Agent များသည် *အတိတ်* မှသင်ယူပေးကာ *ယနေ့* တွင် သင့်တော်သည့် မှတ်ချက်များ ထောက်ပံ့ပေးပြီး *အနာဂတ်* အတွက် ညှိနှိုင်းနိုင်ရန်။  
- **အဓိက**: မသေချာမှုရှိနိုင်ခြေအား လက်ခံသော်လည်း ထင်ရှားမှန်ကန်မှုနှင့် အသုံးပြုသူ ထိန်းချုပ်မှုဖြင့် ယုံကြည်စိတ်ချမှု တည်ဆောက်ရန်။

## 5. အဓိက သင်ခန်းစာများ အကျဉ်းချုပ်

- **သင်ခန်းစာ 1**: Agent များသည် system များဖြစ်ပြီး မော်ဒယ်များတင်သာ မဟုတ်ပါ။ ၎င်းတို့သည် တုံ့ပြန်ခြင်း၊ အကြံဥာဏ်ချုပ်ခြင်းနှင့် လုပ်ဆောင်ခြင်းအား ပြုလုပ်သည်။  
- **သင်ခန်းစာ 2**: Microsoft Agent Framework သည် ကိရိယာခေါ်ယူခြင်းနှင့် state စီမံခန့်ခွဲမှု စိတ်ရှုပ်မှုများကို ပိတ်ပင်ထားသည်။  
- **သင်ခန်းစာ 3**: ပြင်းထန်ပြတ်သားမှုနှင့် အသုံးပြုသူထိန်းချုပ်မှုကို ဦးစားပေးပြီး ဒီဇိုင်းရေးဆွဲရန်။  
- **သင်ခန်းစာ 4**: ကိရိယာများသည် agent ၏ "လက်များ" ဖြစ်သည်။ Schema သတ်မှတ်ခြင်းသည် LLM အတွက် ထိုကိရိယာများကို နားလည်အသုံးပြုရန် အရေးကြီးသည်။  
- **သင်ခန်းစာ 7**: စီမံကိန်းရေးဆွဲခြင်းသည် agent ၏ "အမှုဆောင်လုပ်ငန်း" ဖြစ်ပြီး ခက်ခဲသော workflow များကို ဖြေရှင်းနိုင်စေသည်။
