# AI 代理入門 - 學習指南與課程總結

本指南提供「AI 代理入門」課程的總結，並解釋建立 AI 代理的關鍵概念、框架與設計模式。

## 1. AI 代理介紹

**什麼是 AI 代理？**  
AI 代理是透過賦予大型語言模型（LLM）**工具**、**知識**及**記憶**來擴展其功能的系統。與僅根據訓練資料生成文本的標準 LLM 聊天機器人不同，AI 代理能夠：  
- **感知**其環境（透過感測器或輸入）。  
- **推理**如何解決問題。  
- **行動**以改變環境（透過執行器或工具執行）。

**代理的關鍵組成：**  
- **環境**：代理運行的空間（例如訂票系統）。  
- **感測器**：收集信息的機制（例如呼叫 API）。  
- **執行器**：執行行動的機制（例如發送電子郵件）。  
- **大腦（LLM）**：進行規劃並決定採取何種行動的推理引擎。

## 2. 代理框架

本課程使用 **Microsoft Agent Framework (MAF)** 結合 **Azure AI Foundry Agent Service V2** 來構建代理：

| 組件 | 重點 | 適用於 |
|-----------|-------|----------|
| **Microsoft Agent Framework** | 提供一體化的 Python/C# SDK 用於代理、工具與工作流程 | 建立具備工具、多代理工作流程及生產模式的代理。 |
| **Azure AI Foundry Agent Service** | 托管雲端執行環境 | 提供具備內建狀態管理、可觀察性與信任的安全且可擴展部署。 |

## 3. 代理設計模式

設計模式幫助架構代理的運作方式，以可靠地解決問題。

### **工具使用模式** (課程第4課)  
此模式讓代理能與外部世界互動。  
- **概念**：代理獲得「schema」（可用函數及參數清單）。LLM 根據用戶請求決定*呼叫哪個工具*以及*使用哪些參數*。  
- **流程**：用戶請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM（帶入工具輸出）-> 最終回應。  
- **使用案例**：擷取即時數據（天氣、股票價格）、執行計算、執行程式碼。

### **規劃模式** (課程第7課)  
此模式使代理能解決複雜且多階段的任務。  
- **概念**：代理將一個高階目標拆解成一連串的子任務。  
- **方法**：  
  - **任務分解**：將「規劃旅行」拆分為「訂機票」、「訂飯店」、「租車」。  
  - **反覆規劃**：根據先前步驟的結果重新評估計劃（例如機票已滿，改訂其他日期）。  
- **實作**：通常包含一個「規劃者」代理生成結構化計劃（如 JSON），再由其他代理執行。

## 4. 設計原則

設計代理時需考量三個維度：  
- **空間**：代理應串連人與知識，保持可存取性且不打擾使用者。  
- **時間**：代理應從*過去*學習，於*現在*提供適切提醒，並為*未來*進行調整。  
- **核心**：接受不確定性，但透過透明度及使用者控制建立信任。

## 5. 重點課程摘要

- **第1課**：代理是系統，不只是模型。它們能感知、推理並行動。  
- **第2課**：Microsoft Agent Framework 抽象化了工具呼叫與狀態管理的複雜性。  
- **第3課**：設計時要兼顧透明度與使用者控制。  
- **第4課**：工具是代理的「手」，schema 定義對 LLM 理解如何使用工具至關重要。  
- **第7課**：規劃是代理的「執行功能」，讓代理能處理複雜工作流程。
